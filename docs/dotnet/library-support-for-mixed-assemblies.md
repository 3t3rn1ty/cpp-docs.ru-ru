# <a name="library-support-for-mixed-assemblies"></a>Поддержка библиотек для смешанных сборок

Visual C++ поддерживает использование стандартной библиотеки C++, библиотеку времени выполнения C (CRT), ATL и MFC для приложений, скомпилированных с [/CLR (компиляция CLR)](../build/reference/clr-common-language-runtime-compilation.md). Это позволяет существующих приложений, использующих эти библиотеки для использования также компоненты .NET Framework.

> [!IMPORTANT]
> **/CLR: pure** и **/CLR: safe** параметры компилятора являются устаревшими в Visual Studio 2015 и не поддерживается в Visual Studio 2017 г.

Эта поддержка включает следующие библиотеки DLL и импорта:

- Msvcmrt [d] .lib при компиляции с **/CLR**. Смешанные сборки ссылку на эту библиотеку импорта.

Эта поддержка обеспечивает некоторые дополнительные преимущества:

- CRT и стандартной библиотеки C++ доступны для смешанного кода. CRT и стандартной библиотеки C++ указано не подлежат проверке; в конечном счете вызовы перенаправляются на один и тот же CRT и стандартной библиотеки C++ с помощью из машинного кода.

- Правильная унифицированная обработка исключений в смешанном изображения.

- Статическая инициализация переменных C++ в смешанном изображения.

- Поддержка переменных на уровне домена приложения и на уровне процесса в управляемом коде.

- Это обновление устраняет проблемы блокировки загрузчика, которые применены к смешанные библиотеки DLL, скомпилированные в Visual Studio 2003 и более ранних версий.

Кроме того эта поддержка характерны следующие ограничения:

- Поддерживаются только модели CRT DLL для кода, компилируемого с **/CLR**. Нет статических библиотек CRT, поддерживающих **/CLR** сборки.

Вашей среде (CLR) следует обновить до текущей версии, как работать с более ранними версиями не гарантируется. Код, созданного с помощью этих изменений не будет выполняться в среде CLR версии 1.x.
