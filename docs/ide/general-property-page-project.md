# <a name="general-property-page-project"></a>Страница свойств "Общие" (проект)

Когда щелкните правой кнопкой мыши на узле проекта в обозревателе решений и выберите **свойства**, **Общие** страница свойств под **свойства конфигурации** узел в Левая панель содержит два раздела свойств:

- Общие

- Значения по умолчанию для проекта

Для проектов, отличных от Windows, в разделе [ссылка на страницу свойств C++ Linux](../linux/prop-pages-linux.md)<!-- or [C++ Cross Platform Property Page Reference](../linux/prop-pages-linux.md)-->.

## <a name="general"></a>Общие

Свойства в разделе «Общие» влияют на расположение файлов, создаваемых в процессе построения, а какие файлы удаляются при **Очистить** параметр (**построения** меню) установлен.

**Целевая платформа**  
Задает платформу, в которой этот проект будет выполняться. Например, Windows, Android или iOS. Значение **Windows 10** означает, что проект настроен на универсальную платформу Windows. Если вы используете более ранних версиях Windows, версия не указывается, и значение в этом поле отображается точно так же, как **Windows**. Это доступное только для чтения поле, которое задается при создании проекта.

**Версия пакета SDK для Windows**  
Для целевой платформы Windows указывает версию пакета SDK Windows, необходимую для проекта. При установке C++ рабочей нагрузки с помощью установщика Visual Studio, также устанавливаются необходимые части пакета Windows SDK. Если у вас есть другие версии пакета SDK для Windows на компьютере, в раскрывающемся списке отображается каждая версия средств пакета SDK, которые вы установили.

Чтобы ориентироваться на Windows 7 или Windows Vista, используйте значение **8.1**, поскольку Windows SDK 8.1 имеет обратную совместимость с этими платформами. Кроме того, необходимо определить соответствующее значение параметру **_WIN32_WINNT** в targetver.h. Для Windows 7 оно равно 0x0601. В разделе [изменение WINVER и _WIN32_WINNT](../porting/modifying-winver-and-win32-winnt.md).

Вы можете установить набор инструментов платформы Windows XP, включенный в Visual Studio, чтобы использовать текущую версию библиотек для построения проектов Windows XP и Windows 2003 Server. Сведения о получении и использовании этого набора инструментов платформы см. в разделе [Настройка программ для Windows XP](../build/configuring-programs-for-windows-xp.md). Дополнительную информацию об изменении набора инструментов платформы см. в разделе [Практическое руководство. Изменение требуемой версии .NET Framework и набора средств платформы](../build/how-to-modify-the-target-framework-and-platform-toolset.md).

**Минимальная версия целевой платформы**  
Указывает самую раннюю версию платформы, где проект может работать. Это свойство отображается только в том случае, если тип проекта поддерживает его, например, в проектах универсальной платформы Windows. Если приложение может воспользоваться преимуществами функций в более новых версиях пакета SDK для Windows, однако при этом может запускаться в более ранних версиях без таких функций с некоторой возможной потерей функциональности, то значения этих свойств могут отличаться. В этом случае код должен во время выполнения проверять версию платформы, где он запущен, и не должен пытаться использовать функции, недоступные в старой версии платформы.

Обратите внимание, что Visual C++ не использует этот параметр. Он включен для согласованности с другими языками, такими как C# и JavaScript, и в качестве подсказки для тех, кто использует ваш проект. Visual C++ не выдает ошибку при использовании возможности, недоступной в минимально возможной версии.

**Выходной каталог**  
Указывает каталог, куда инструменты, такие как компоновщик и другие, будут помещать конечные выходные файлы, созданные в процессе сборки. Как правило, к ним относятся такие инструменты вывода данных, как компоновщик, библиотекарь или BSCMake. По умолчанию это свойство является каталог, указанный с помощью $ $(SolutionDir) макросы (конфигурация) \.

Для программного доступа к этому свойству см. раздел <xref:Microsoft.VisualStudio.VCProjectEngine.VCConfiguration.OutputDirectory%2A>.

**Промежуточный каталог**  
Указывает каталог, куда инструменты, такие как компилятор, будут помещать все промежуточные файлы, созданные в процессе сборки. Как правило, к ним относятся такие инструменты вывода данных, как компилятор C/C++, MIDL и компилятор ресурсов. По умолчанию это свойство является каталог, указанный с помощью макроса $(конфигурация) \.

Для программного доступа к этому свойству см. раздел <xref:Microsoft.VisualStudio.VCProjectEngine.VCConfiguration.IntermediateDirectory%2A>.

**Целевое имя**  
Задает имя файла, который создает этот проект. По умолчанию это свойство является именем, указанным в макрос $(ProjectName).

**Конечное расширение**  
Указывает расширение имени файла, которое создает этот проект, например DLL или EXE.

**Расширения для удаления при очистке**  
**Очистить** параметр (**построения** меню) удаляет файлы из промежуточного каталога, где создается конфигурация проекта. Файлы с расширениями, указанными с помощью этого свойства будут удалены при **Очистить** или при выполнении повторного построения. Кроме файлов с такими расширениями в промежуточном каталоге, система сборки также удаляет все известные выходные данные сборки независимо от того, где они расположены (включая промежуточные выходные данные, такие как файлы OBJ). Обратите внимание, что можно указать подстановочные знаки.

Для программного доступа к этому свойству см. раздел <xref:Microsoft.VisualStudio.VCProjectEngine.VCConfiguration.DeleteExtensionsOnClean%2A>.

**Файл журнала сборки**  
Позволяет указать нестандартное расположение для файла журнала, создаваемого при каждой сборке проекта. Расположение по умолчанию указывается .log макросы $(IntDir) $(MSBuildProjectName).

Вы можете использовать макросы проекта, чтобы изменить расположение этого каталога. В разделе [общие макросы для команд и свойств построения](../ide/common-macros-for-build-commands-and-properties.md).

**Набор инструментов платформы**  
Позволяет проекту ориентироваться на другую версию компилятора и библиотек Visual C++. Проекты Visual C++ можно ориентироваться либо по умолчанию набор инструментов устанавливается Visual Studio или один из наборов инструментов, установленный в несколько предыдущих версиях Visual Studio, включая наборы инструментов, создание исполняемых объектов, которые можно запускать в Windowx XP. Сведения об изменении набора инструментов платформы см. в разделе [как: Изменение требуемой версии .NET Framework и набора инструментов платформы](../build/how-to-modify-the-target-framework-and-platform-toolset.md).

**Включить управляемую добавочную сборку**  
Для управляемых проектов это позволяет определить внешний доступ при создании сборки. Изменение управляемого проекта не является видимым для других проектов, зависимые проекты не будут перестроены. Это может значительно ускорить сборку в решениях, включающие управляемые проекты.

## <a name="project-defaults"></a>Значения по умолчанию для проекта

В разделе "Проект по умолчанию" представлены свойства по умолчанию, которые можно изменять. Определения для этих свойств можно найти в файлах с расширением PROPS в *каталог установки*\VC\VCProjectDefaults.

**Тип конфигурации**  
Для выбора доступно несколько типов конфигурации:

- **Приложение (.exe)**, отображается набор инструментов компоновщика (компилятор C/C++, MIDL, компилятор ресурсов, компоновщик, BSCMake, создания прокси-сервера веб-служб XML, события настраиваемой сборки, перед сборкой, перед компоновкой и после сборки).

- **Динамическая библиотека (.dll)**, отображается набор инструментов компоновщика, задает параметр компоновщика/DLL и добавляет определение _WINDLL в CL.

- **Makefile**, отображает набор инструментов makefile (NMake).

- **Статическая библиотека (.lib)**, отображает набор инструментов библиотекаря (идентичен набору инструментов компоновщика, но вместо компоновщика используется библиотекарь и создания прокси-сервера веб-служб XML).

- **Программа**, отображается набор служебных программ (MIDL, события после сборки, события настраиваемой сборки,).

Для программного доступа к этому свойству см. раздел <xref:Microsoft.VisualStudio.VCProjectEngine.VCConfiguration.ConfigurationType%2A>.

**Использование MFC**  
Указывает, будет ли проект MFC статически или динамически связан с библиотекой DLL MFC. Можно выбрать проекты MFC не **использовать стандартные библиотеки Windows** для связи с различными библиотеками Win32, которые включаются при использовании MFC.

Для программного доступа к этому свойству см. раздел <xref:Microsoft.VisualStudio.VCProject.VCProjectConfigurationProperties.useOfMfc%2A>.

**Использование ATL**  
Указывает, будет ли проект MFC статически или динамически связан с библиотекой DLL ATL. Если указать любое значение, отличное от **без использования ATL**, компилятора будет добавлено определение **командной строки** страницу свойств.

Для программного доступа к этому свойству см. раздел <xref:Microsoft.VisualStudio.VCProject.VCProjectConfigurationProperties.useOfATL%2A>.

**Набор символов**  
Определяет, следует ли задать _UNICODE или _MBCS. Кроме того, влияет на точку входа компоновщика, если это уместно.

Для программного доступа к этому свойству см. раздел <xref:Microsoft.VisualStudio.VCProject.VCProjectConfigurationProperties.CharacterSet%2A>.

**Поддержка среды CLR**  
Вызывает [/CLR](../build/reference/clr-common-language-runtime-compilation.md) параметр компилятора для использования.

Для программного доступа к этому свойству см. раздел <xref:Microsoft.VisualStudio.VCProject.VCProjectConfigurationProperties.ManagedExtensions%2A>.

**Целевой версии платформы .NET**  
В управляемых проектах указывает версии платформы .NET framework, целевой объект.

**Оптимизация всей программы**  
Указывает [/GL](../build/reference/gl-whole-program-optimization.md) параметр компилятора и [/LTCG](../build/reference/ltcg-link-time-code-generation.md) компоновщика. По умолчанию это отключить для конфигураций отладки и включен для розничной торговли конфигураций.

**Поддержка приложений для магазина Windows**  
Указывает, поддерживает ли этот проект приложения для магазина Windows. Дополнительные сведения см. в разделе [/ZW (компиляция среды выполнения Windows)](../build/reference/zw-windows-runtime-compilation.md)и Центр разработчиков Windows.
