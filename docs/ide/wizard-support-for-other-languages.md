# <a name="wizard-support-for-other-languages"></a>Поддержка мастера для других языков
При установке Visual Studio программа установки определяет язык и региональные параметры системы и устанавливает соответствующий языковой шаблон или шаблоны для данного языкового стандарта. Например, для западноевропейских языковых стандартов программа установки устанавливает английский, французский, итальянский, испанский и немецкий языки. Эти языки отображаются в списке **Язык ресурсов** на странице [Тип приложения](../mfc/reference/application-type-mfc-application-wizard.md) мастера приложений MFC.  
  
## <a name="language-templates"></a>Языковые шаблоны  
 Не все шаблоны устанавливаются во всех системах, так как шаблоны основаны на кодировке ANSI и не все ресурсы можно редактировать во всех системах. Например, по умолчанию невозможно редактировать японские ресурсы во французской системе.  
  
 Если вы используете Windows 2000 или более поздней версии и хотите создать приложение MFC на другом языке, нужно скопировать каталог шаблонов для соответствующего языка с установочного носителя Visual Studio (диск 1) в систему.  
  
> [!NOTE]
>  Чтобы изменить созданный проект, нужно изменить язык системы на подходящее значение для выбранного языка.  
  
 Каждый из шаблонов назначается папке в каталоге \Microsoft Visual Studio .NET 2003\Vc7\VCWizards\mfcappwiz\templates\, как указано в таблице ниже. Чтобы получить доступ к языковому шаблону, скопируйте соответствующую папку в каталог \mfcappwiz\templates\ на своем компьютере. После копирования папки этот язык отображается в списке **Язык ресурсов** на странице **Тип приложения** мастера приложений MFC.

## <a name="format-of-visual-c-wizard-generated-files"></a>Формат файлов, созданных мастером Visual C++  
 Если установленная языковая версия Visual Studio не совпадает с языком системы, мастеры Visual C++ будут создавать проекты в кодировке Юникод. Например, при установке японской версии Visual Studio на компьютере с региональными параметрами, заданными для любого другого языка, мастеры Visual C++ создадут проекты, состоящие из файлов в кодировке Юникод. Это типичная ситуация на компьютерах с пакетами многоязыкового интерфейса (MUI) Windows.  
  
 Это поведение отличается от систем, настроенных таким образом, чтобы язык системы совпадал с языковой версией Visual Studio. В этом случае файлы проекта будут создаваться в формате ANSI с системной кодовой страницей.
