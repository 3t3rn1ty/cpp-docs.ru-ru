# <a name="specifying-build-events"></a>Задание событий построения
События сборки можно использовать для указания команды, которые выполняются до начала построения, до процесса компоновки или после завершения построения.  
  
 События сборки выполняются, только если сборка успешно достигает этих точек в процессе сборки. При возникновении ошибки в сборке, *после построения* событие не происходит; Если эта ошибка возникает до фазы компоновки, ни *перед компоновкой* ни *после построения* будет событий возникать. Кроме того, если файлы не должен быть связан, *перед компоновкой* событие не происходит. *Перед компоновкой* событий не доступна в проектах, которые не содержат этапа компоновки.  
  
 Если отсутствуют файлы, необходимые для построения, события построения не возникнет.  
  
 Общие сведения о событиях построения см. в разделе [основные сведения о пользовательских сборки действия и события построения](../ide/understanding-custom-build-steps-and-build-events.md).  
  
### <a name="to-specify-a-build-event"></a>Чтобы указать событие сборки  
  
1.  В **обозревателе решений** выберите проект, для которого необходимо задать событие сборки.  
  
2.  Откройте диалоговое окно **Страницы свойств** проекта. Дополнительные сведения см. в разделе [работа со свойствами проекта](../ide/working-with-project-properties.md).  
  
3.  В **события построения** папку, выберите страницу свойств события построения.  
  
4.  Укажите свойства, связанные с событием сборки:  
  
    -   В **командной строки**, указать команду, как если бы она использовалась в командной строке. Укажите допустимую команду или пакетного файла и любые нужные входные данные или выходные файлы. Укажите **вызовите** пакетной команды перед именем пакетного файла, чтобы обеспечить выполнение всех последующих команд.  
  
         Несколько входных и выходных файлов можно указать с помощью макросов MSBuild символически. [!INCLUDE[crabout](../build/reference/includes/crabout_md.md)]Указание местоположения файлов или имена наборов файлов, см. [общие макросы для команд и свойств построения](../ide/common-macros-for-build-commands-and-properties.md).  
  
         Так как при указании переменной среды необходимо заменить каждый символ «%» зарезервирован MSBuild,  **%**  escape-символ **% 25** Шестнадцатеричная escape-последовательность. Например, замените **% WINDIR %** с **% 25WINDIR % 25**. MSBuild заменяет каждый **% 25** последовательность с  **%**  символов перед обращением к переменной среды.  
  
    -   В **описание**, введите описание для этого события. Описание будет выводиться в **вывода** окна при возникновении этого события.  
  
    -   В **исключено из построения**, укажите **Да** Если не нужно, чтобы событие для запуска.
