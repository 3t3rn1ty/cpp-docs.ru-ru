# Разделители для тегов документации Visual C++
[!INCLUDE[vs2017banner](../assembler/inline/includes/vs2017banner.md)]

Использование тегов документации для разделителей, которые указывают на данные, комментарий документации начинается и заканчивается.  
  
 С тегами в XML\-документации можно использовать следующие виды разделителей:  
  
 `///`  
 Это форма, отображаемой в примерах документации и используется в шаблонах проектов C Visual C\+\+.  
  
 `/** */`  
 Эти многополосные разделителей.  
  
 Правила форматирования при использовании разделителей `/** */`:  
  
-   Для линии содержащий разделитель `/**`, если остаток линии пробел, линия не обрабатывается для комментариев.  Если первый символ пробела, этот пробелы игнорируются и обрабатываются остальные линии.  В противном случае весь текст, расположенный в строке после разделителя `/**`, обрабатывается как часть комментария.  
  
-   Для линии содержащий разделитель `*/`, если только пробелы до разделителя `*/`, что линия, игнорируется.  В противном случае текст строки, расположенный перед разделителем `*/`, обрабатывается как часть комментария, и к нему применяются правила сопоставления шаблонов, описанные в следующем разделе.  
  
-   Для линии после одно, которое начинается с разделителем `/**` компилятор ищет общий шаблон в начале каждой линии, состоящая из дополнительного места и звездочки \(`*`\) и два более необязательным пробелом.  Если компилятор обнаруживает общий набор символов в начале каждой линии, то он игнорирует этот шаблон для всех линий после разделителя `/**` и до и, возможно, включая линия, содержащий разделитель `*/`.  
  
 Некоторые примеры.  
  
-   В следующем комментарии будет обработана только строка, которая начинается с `<summary>`.  Следующие 2 формата тега предоставляют одинаковые комментарии:  
  
    ```  
    /**  
    <summary>text</summary>   
    */  
    /** <summary>text</summary> */  
    ```  
  
-   Компилятор применяет шаблон "\*", чтобы пропустить в начале второго и третьего линий.  
  
    ```  
    /**  
     * <summary>  
     *  text </summary>*/  
    ```  
  
-   Компилятор не найден ни один шаблон в этом комментарии из\-за отсутствия звездочка на второй линии.  Следовательно, весь текст на второго и третьего линиям, устанавливаемых до `*/`, будут обрабатываться как часть комментария.  
  
    ```  
    /**  
     * <summary>  
       text </summary>*/  
    ```  
  
-   Компилятор не найден ни один шаблон в этом комментарии по двум причинам.  Во\-первых, линия, которая начинается с последовательным числом перерывов перед звездочкой.  Во\-вторых, пятая строка начинается с символа табуляции, который отличается от пробелов.  Следовательно, весь текст из второй линии до тех пор, пока `*/` не будет представлен как часть комментария.  
  
    ```  
    /**  
      * <summary>  
      * text   
     *  text2  
       *  </summary>  
    */  
    ```
