# <a name="2723-lastprivate"></a>2.7.2.3 lastprivate
`lastprivate` Предложение предоставляет надмножество функциональные возможности, предоставляемые `private` предложения. Синтаксис `lastprivate` предложение является следующим образом:  
  
```  
lastprivate(variable-list)  
```  
  
 Переменные, указанные в *списка переменной* имеют `private` семантику предложения. Когда `lastprivate` присутствует предложение в директиве, определяющая конструкции совместной работы, значение каждого `lastprivate` присваивается переменной из последовательно последней итерации связанного цикла или директиве лексически последний раздел исходный объект этой переменной. Переменные, которые не являются значение которой назначено при последней итерации **для** или **параллельных для**, или лексически последний раздел **разделы** или  **Параллельные разделы** директивы, имеют неопределенные значения после конструкции. Неназначенные подчиненных также имеют неопределенное значение после конструкции.  
  
 Ограничения, `lastprivate` предложение, следующим образом:  
  
-   Все ограничения для `private` применения.  
  
-   Переменная с типом класса, который указан как `lastprivate` должен иметь доступный и однозначный оператор присваивания копии.  
  
-   Переменные, которые закрытый внутри параллельной области или, входящие в `reduction` предложения **параллельных** директива не может быть указана в `lastprivate` предложение в директиве совместной работы, который привязывается к параллельной конструкции.
