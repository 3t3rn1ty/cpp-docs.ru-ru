# <a name="2725-default"></a>2.7.2.5 default
**По умолчанию** предложение пользователь может повлиять на общий доступ к данным, атрибуты переменных. Синтаксис **по умолчанию** предложение является следующим образом:  
  
```  
default(shared | none)  
```  
  
 Указание **default(shared)** эквивалентен явное перечисление каждой переменной, видимых в настоящее время в **общего** предложение, если он не является **threadprivate** или **против**`t`-полное. В случае отсутствия явно **по умолчанию** предложение, поведение по умолчанию является таким же, как если **default(shared)** были указаны.  
  
 Указание **default(none)** по крайней мере одно из следующих должен иметь значение true для каждую ссылку на переменную в лексическую область параллельной конструкции:  
  
-   Переменная явно перечислены в предложении совместного использования данных атрибута конструкции, которая содержит ссылку.  
  
-   Переменная объявлена в параллельной конструкции.  
  
-   Переменная **threadprivate**.  
  
-   Переменная имеет **const**-типа с указанием.  
  
-   Переменная является управляющей переменной цикла для **для** цикл, который следует сразу за **для** или **параллельных для** директивы и ссылка на переменную появляется внутри цикла .  
  
 Задание переменной на **firstprivate**, **lastprivate**, или **сокращения** предложения директивы заключенными вызывает неявную ссылку на переменную в заключающей контекст. Такие неявные ссылки также подлежат указанным выше требованиям.  
  
 Только один **по умолчанию** может быть указано предложение на **параллельных** директивы.  
  
 Значения переменных по умолчанию атрибутов совместного использования данных можно переопределить с помощью **закрытый**, **firstprivate**, **lastprivate**, **сокращения**, и **общего** предложений, как показано в следующем примере:  
  
```  
#pragma  omp  parallel  for  default(shared)  firstprivate(i)\  
   private(x)  private(r)  lastprivate(i)  
```
