# <a name="2721-private"></a>2.7.2.1 private
`private` Предложение объявляет переменные в списке переменной должно быть закрытым для каждого потока в команде. Синтаксис `private` предложение является следующим образом:  
  
```  
private(variable-list)  
```  
  
 Поведение переменной, указанной в `private` предложение является следующим образом. Для конструкции объекта с автоматической длительностью хранения выделяется. Размер и выравнивание нового объекта определяется тип переменной. Такое распределение происходит один раз для каждого потока в группе и вызывается конструктор по умолчанию для объекта класса, при необходимости; в противном случае начальное значение не определено.  Исходный объект ссылается переменная имеет неопределенное значение после входа конструкции, не должны изменяться в пределах области динамических конструкции и имеет неопределенное значение после выхода из конструкции.  
  
 В лексической области директив конструкции переменная ссылается на новый закрытый объект, выделенный в потоке.  
  
 Ограничения, `private` предложение, следующим образом:  
  
-   Переменная с типом класса, который указан в `private` предложение должен иметь конструктор по умолчанию, доступный и однозначный.  
  
-   Переменная, указанная в `private` не должны иметь предложение **const**-указанием типа, если оно не имеет типа, имеющего класс `mutable` член.  
  
-   Переменная, указанная в `private` предложение не должны иметь неполный тип или ссылочный тип.  
  
-   Переменные, которые отображаются в `reduction` предложения **параллельных** директива не может быть указана в `private` предложение в директиве совместной работы, который привязывается к параллельной конструкции.
