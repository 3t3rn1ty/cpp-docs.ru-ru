# <a name="atl-control-wizard"></a>Мастер элементов управления ATL
Вставки в проект ATL (или проект MFC с поддержкой ATL) элемента управления ATL. Этот мастер можно использовать для вставки трех видов элементов управления:  
  
-   Стандартный элемент управления  
  
-   Составной элемент управления  
  
-   Элемент управления DHTML  
  
 Кроме того, можно указать минимальный элемент управления удаление интерфейсы из [интерфейсы](../../atl/reference/interfaces-atl-control-wizard.md) списка, в котором приведены в качестве значения по умолчанию для элементов управления открывать в большинстве контейнеров. Можно задать интерфейсы, которые нужно поддерживать для элемента управления в **интерфейсы** мастера.  
  
## <a name="remarks"></a>Примечания  
 Скрипт регистрации, созданные этим мастером зарегистрирует COM-компонентов в разделе HKEY_CURRENT_USER вместо HKEY_LOCAL_MACHINE. Чтобы изменить это поведение, присвойте **компонент регистра для всех пользователей** параметр мастера ATL.  
  
## <a name="names"></a>Имена  
 Укажите имена объекта, интерфейса и классов, которые добавлены в проект. За исключением **короткое имя**, все поля, которые могут изменяться независимо друг от друга. Если изменить текст для **короткое имя**, это изменение отражается в имена всех полей на этой странице. При изменении **Coclass** имя в разделе COM, изменение отражается в **тип** поле, но **интерфейс** имя и **ProgID** не изменяются. Эти принципы именования позволяют легко распознать объекты по вы все имена разработки элементов управления.  
  
> [!NOTE]
>  **Компонентный класс** можно изменять, только без атрибутов элементов управления. Если атрибуты проекта, нельзя изменить **компонентный класс**.  
  
### <a name="c"></a>C++  
 Содержит сведения о классе C++, созданном для реализации объекта.  
  
 **Краткое имя**  
 Задает сокращенное имя для объекта. Указываемое имя определяет класс и **Coclass** имя файла (. CPP и. (H) имена, имя интерфейса и **тип** имена, если не изменить значения этих полей отдельно.  
  
 **Класс**  
 Задает имя класса, реализующего объект. Это имя основано на имени, введенного в **короткое имя**, предшествует «C» — типичный префикс имени класса.  
  
 **h-файл**  
 Задает имя файла заголовка для нового класса объекта. По умолчанию это имя основано на имени, введенного в **короткое имя**. Нажмите кнопку с многоточием, чтобы сохранить файл в выбранное расположение или добавить объявление класса в существующий файл. Если выбран существующий файл, не он будет сохранен в выбранном месте пока вы щелкните **Готово**.  
  
 Мастер не перезаписывает файл. Если выбрать имя существующего файла, после нажатия **Готово**, мастер попросит указать, следует ли добавить объявление класса к содержимому файла. Щелкните **Да** для добавления файла; щелкните **нет** вернуться в мастер и укажите другое имя файла.  
  
 **CPP-файл**  
 Задает имя файла реализации класса нового объекта. По умолчанию это имя основано на имени, введенного в **короткое имя**. Нажмите кнопку с многоточием, чтобы сохранить файл в выбранное расположение. Файл не сохраняется в выбранном месте до нажатия **Готово** в мастере.  
  
 Мастер не перезаписывает файл. Если выбрать имя существующего файла, после нажатия **Готово**, мастер попросит указать, следует ли добавить реализацию класса к содержимому файла. Щелкните **Да** для добавления файла; щелкните **нет** вернуться в мастер и укажите другое имя файла.  
  
 **Атрибуты**  
 Указывает, использует ли объект атрибуты. При добавлении объекта в атрибутами проект ATL, этот параметр установлен и недоступен для изменения. То есть только помеченных атрибутами объектов можно добавить в проект с поддержкой атрибутов.  
  
 Объект с атрибутами можно добавлять только в проект ATL, который использует атрибуты. Если этот флажок для проекта ATL, не поддерживающего атрибуты, мастер попросит указать, следует ли добавить поддержку атрибутов в проект.  
  
 По умолчанию все объекты, которые добавляются после установки этого флажка обозначаются как атрибутивного (флажок). Можно снять этот флажок, чтобы добавить объект, который использует атрибуты.  
  
 В разделе [параметры приложения, мастер проекта ATL](../../atl/reference/application-settings-atl-project-wizard.md) и [основные механизмы атрибуты](../../windows/basic-mechanics-of-attributes.md) подробнее.  
  
### <a name="com"></a>COM  
 Сведения о функциях COM для объекта.  
  
 **Компонентный класс**  
 Задает имя класса компонента, содержащий список интерфейсов, поддерживаемое объектом.  
  
> [!NOTE]
>  Если вы создаете проект, использующий атрибуты, или если на этой странице мастера указывается, что элемент управления использует атрибуты, не может изменить этот параметр, поскольку ATL не включает **coclass** атрибута.  
  
 **Интерфейс**  
 Задает имя интерфейса для объекта. По умолчанию имя интерфейса добавляется символ «I».  
  
 **Тип**  
 Задает описание объекта, которое будет отображаться в реестре  
  
 **Идентификатор progID**  
 Задает имя, которое может использоваться контейнерами вместо CLSID объекта. Это поле не заполняется автоматически. Если это поле не заполняется вручную, элемент управления не может быть недоступен для других средств. Например, элементы управления ActiveX, созданные без `ProgID` , недоступны в **вставить элемент ActiveX** диалоговое окно. Дополнительные сведения о диалоговом окне см. в разделе [вставить элемент управления диалогового окна ActiveX](../../windows/insert-activex-control-dialog-box.md).
