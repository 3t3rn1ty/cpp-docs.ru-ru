# мастер диалоговых окон ATL
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

Этот мастер вставляет в проект объект диалогового окна ATL, производный от [CAxDialogImpl](../Topic/CAxDialogImpl%20Class.md).  Диалоговое окно, производное от `CAxDialogImpl`, может содержать элементы управления ActiveX.  
  
 Мастер создает ресурс диалогового окна с кнопками **ОК** и **Отмена** по умолчанию.  Можно редактировать ресурс диалогового окна и добавлять в него элементы управления ActiveX с помощью [редактора диалоговых окон](../../mfc/dialog-editor.md) в представлении ресурсов.  
  
 Мастер вставляет в файл заголовка [схему сопоставления сообщений](../../atl/message-maps-atl.md) и объявления для обработки событий нажатия клавиши, используемых по умолчанию.  Дополнительные сведения о диалоговых окнах ATL см. в разделе [Реализация диалогового окна](../../atl/implementing-a-dialog-box.md).  
  
 **Краткое имя**  
 Задает сокращенное имя объекта диалогового окна ATL.  Введенное имя определяет имя класса и имена файлов \(CPP и H\), если только не изменить значения этих полей отдельно.  
  
 `Class`  
 Задает имя создаваемого класса.  Это имя основано на имени, указанном в поле **Краткое имя**, перед которым добавляется символ "C" — типичный префикс имени класса.  
  
 **H\-файл**  
 Задает имя файла заголовка для класса нового объекта.  По умолчанию это имя основано на имени, заданном в поле **Краткое имя**.  Нажмите кнопку с символом многоточия, чтобы сохранить файл в указанном месте или добавить объявление класса в существующий файл.  Если выбрать уже существующий файл, он не будет сохранен в выбранном месте до тех пор, пока в мастере не будет нажата кнопка **Готово**.  
  
 Мастер не переписывает файл.  Если выбрать имя существующего файла, после нажатия кнопки **Готово** мастер попросит указать, следует ли добавить объявление класса к имеющемуся содержимому файла.  Чтобы добавить реализацию в файл, нажмите кнопку **Да**; чтобы вернуться в мастер и указать другое имя файла, нажмите кнопку **Нет**.  
  
 **CPP\-файл**  
 Задает имя файла реализации класса нового объекта.  По умолчанию это имя основано на имени, заданном в поле **Краткое имя**.  Нажмите кнопку с символом многоточия, чтобы сохранить файл в указанном месте.  Файл не сохраняется в выбранном месте до тех пор, пока в мастере не будет нажата кнопка **Готово**.  
  
 Мастер не переписывает файл.  Если выбрать имя существующего файла, после нажатия кнопки **Готово**, то мастер попросит указать, следует ли добавить реализацию класса к имеющемуся содержимому файла.  Чтобы добавить реализацию в файл, нажмите кнопку **Да**; чтобы вернуться в мастер и указать другое имя файла, нажмите кнопку **Нет**.
