# <a name="com-10-atl-com-10-component-wizard"></a>COM+ 1.0, мастер компонентов ATL COM+ 1.0
Позволяет указать тип интерфейса и дополнительные интерфейсы для поддержки этой странице мастер компонентов ATL COM + 1.0.  
  
 Дополнительные сведения о проектах ATL и ATL COM-классов см. в разделе [ATL COM Desktop компонентов](../../atl/atl-com-desktop-components.md).  
  
 **Interface**  
 Указывает тип интерфейса, поддерживаемого объектом. По умолчанию объект поддерживает сдвоенный интерфейс.  
  
|Параметр|Описание:|  
|------------|-----------------|  
|**Два двухъядерных**|Указывает, что объект поддерживает сдвоенный интерфейс (его таблица виртуальных методов имеет функции пользовательского интерфейса и позднее связывание `IDispatch` методов). Позволяет COM-клиенты и контроллеры автоматизации, доступ к объекту.|  
|**Пользовательский**|Указывает, что объект поддерживает пользовательский интерфейс (его таблица виртуальных методов имеет функции пользовательского интерфейса). Пользовательский интерфейс может быть быстрее, чем сдвоенный интерфейс, особенно через границы процессов.<br /><br /> -   **Совместимый с автоматизацией** добавляет Поддержка модели автоматизации пользовательского интерфейса. Атрибутами проекты задает **oleautomation** атрибута в компонентном классе.|  
  
 **Использовать очереди**  
 Указывает, что клиенты смогут вызывать этот компонент асинхронно, используя очереди сообщений. Добавляет макрос атрибута компонента custom (TLBATTR_QUEUEABLE, 0) в h-файл (атрибутами проектов) или IDL-файл (без атрибутов проектов).  
  
 **Поддержка**  
 Указывает Дополнительные сведения об ошибке обработки и управления объектом.  
  
|Параметр|Описание:|  
|------------|-----------------|  
|**ISupportErrorInfo**|Добавляет поддержку [ISupportErrorInfo](../../atl/reference/isupporterrorinfoimpl-class.md) интерфейс, может возвращать сведения об ошибке на клиент.|  
|**IObjectControl**|Предоставляет доступ к трем объекта [IObjectControl](http://msdn.microsoft.com/library/windows/desktop/ms686474) методы: [активировать](http://msdn.microsoft.com/library/windows/desktop/ms681303), [CanBePooled](http://msdn.microsoft.com/library/windows/desktop/ms684322), и [деактивировать](http://msdn.microsoft.com/library/windows/desktop/ms687094).|  
|**IObjectConstruct**|Добавляет поддержку [IObjectConstruct](http://msdn.microsoft.com/library/windows/desktop/ms680583) интерфейс передача управления в параметрах из других методов или объектов.|  
  
 **Транзакция**  
 Указывает, что объект поддерживает транзакции. Включает файл mtxattr.h в IDL-файл (без атрибутов проектов).  
  
|Параметр|Описание:|  
|------------|-----------------|  
|**Поддерживается**|Указывает, что объект является никогда не корня потока транзакций, добавляя макрос атрибута компонента custom(TLBATTR_TRANS_SUPPORTED,0) в h-файл (атрибутами проектов) или IDL-файл (без атрибутов проектов).|  
|**Обязательное**|Указывает, что объект может или не может быть корнем потока транзакций, добавляя макрос атрибута компонента custom(TLBATTR_TRANS_REQUIRED,0) в h-файл (атрибутами проектов) или IDL-файл (без атрибутов проектов).|  
|**Не поддерживается**|Указывает, что объект не поддерживает транзакции. Добавляет custom(TLBATTR_TRANS_NOTSUPP,0) макрос атрибута компонента h-файл (атрибутами проектов) или IDL-файл (без атрибутов проектов).|  
|**Требуется новая**|Указывает, что объект всегда корня потока транзакций, добавляя макрос атрибута компонента custom(TLBATTR_TRANS_REQNEW,0) в h-файл (атрибутами проектов) или IDL-файл (без атрибутов проектов).|
