# <a name="atl-com-10-component-wizard"></a>Мастер компонентов ATL COM+ 1.0
Этот мастер используется для добавления объекта к проекту, поддерживающего службы COM + 1.0, включая транзакции.  
  
 Можно указать, поддерживает ли объект сдвоенные интерфейсы и автоматизацию. Также можно задать поддержку интерфейса информации об ошибках, расширенного управления объектами, транзакций и асинхронных очередей сообщений.  
  
## <a name="remarks"></a>Примечания  
 Начиная с версии [!INCLUDE[vs_orcas_long](../../atl/reference/includes/vs_orcas_long_md.md)], скрипт регистрации, созданные этим мастером зарегистрирует COM-компонентов в группе **HKEY_CURRENT_USER** вместо **HKEY_LOCAL_MACHINE**. Чтобы изменить это поведение, присвойте **компонент регистра для всех пользователей** параметр мастера ATL.  
  
## <a name="names"></a>Имена  
 Укажите имена объекта, интерфейса и классов, которые добавлены в проект. За исключением класса **короткое имя**, всех прочих полях можно менять независимо от других. Если изменить текст для **короткое имя**, это изменение отражается в имена всех полей на этой странице. При изменении **Coclass** имя в разделе COM, изменение отражается в **тип** и **ProgID** поля, но **интерфейс** имя не изменяется. Эти принципы именования позволяют легко распознать объекты по вы все имена разработки элементов управления.  
  
 **Краткое имя**  
 Задает сокращенное имя для объекта. Имя, которое определяет `Class` и `Coclass` имена, **CPP-файл** и **h-файл** имена, **интерфейс** имя **тип** имена и **ProgID**, если не изменить значения этих полей отдельно.  
  
 **h-файл**  
 Задает имя файла заголовка для нового класса объекта. По умолчанию это имя основано на имени, введенного в **короткое имя**. Нажмите кнопку с многоточием, чтобы сохранить файл в выбранное расположение или добавить объявление класса в существующий файл. Если выбран существующий файл, не он будет сохранен в выбранном месте пока вы щелкните **Готово** в мастере.  
  
 Мастер не перезаписывает файл. Если выбрать имя существующего файла, после нажатия **Готово**, мастер попросит указать, следует ли добавить объявление класса к содержимому файла. Щелкните **Да** для добавления файла; щелкните **нет** вернуться в мастер и укажите другое имя файла.  
  
 **Класс**  
 Задает имя создаваемого класса. Это имя основано на имени в **короткое имя**, предшествует «C» — типичный префикс имени класса.  
  
 **CPP-файл**  
 Задает имя файла реализации класса нового объекта. По умолчанию это имя основано на имени в **короткое имя**. Нажмите кнопку с многоточием, чтобы сохранить файл в выбранное расположение. Файл не сохраняется в выбранном месте до нажатия **Готово** в мастере.  
  
 Мастер не перезаписывает файл. Если выбрать имя существующего файла, после нажатия **Готово**, мастер попросит указать, следует ли добавить реализацию класса к содержимому файла. Щелкните **Да** для добавления файла; щелкните **нет** вернуться в мастер и укажите другое имя файла.  
  
 **Атрибуты**  
 Указывает, использует ли объект атрибуты. При добавлении объекта в атрибутами проект ATL, этот параметр установлен и недоступен для изменения. То есть только помеченных атрибутами объектов можно добавить в проект с поддержкой атрибутов.  
  
 Если этот флажок для проекта ATL, не поддерживающего атрибуты, мастер попросит указать, следует ли добавить поддержку атрибутов в проект.  
  
 Любые объекты, которые добавляются после установки этого параметра определяются как атрибуты по умолчанию (флажок). Можно снять этот флажок, чтобы добавить объект, который использует атрибуты.  
  
 В разделе [параметры приложения, мастер проекта ATL](../../atl/reference/application-settings-atl-project-wizard.md) и [основные механизмы атрибуты](../../windows/basic-mechanics-of-attributes.md) подробнее.  
  
### <a name="com"></a>COM  
 Сведения о функциях COM для объекта.  
  
 **Компонентный класс**  
 Задает имя класса компонента, содержащий список интерфейсов, поддерживаемое объектом.  
  
> [!NOTE]
>  Если вы создаете проект, использующий атрибуты, или если на этой странице мастера указывается, что компонент COM + 1.0 использует атрибуты, не может изменить этот параметр, поскольку ATL не включает `coclass` атрибута.  
  
 **Тип**  
 Задает описание объекта, которое будет отображаться в реестре  
  
 **Интерфейс**  
 Задает интерфейс, создаваемый для объекта. Этот интерфейс содержит пользовательские методы.  
  
 **Идентификатор progID**  
 Задает имя, которое может использоваться контейнерами вместо CLSID объекта.
