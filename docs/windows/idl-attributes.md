# <a name="idl-attributes"></a>Атрибуты IDL

В большинстве случаев обслуживание IDL-файле означало, что приходилось:

- Уметь работать с структуры и синтаксиса IDL-файле, чтобы иметь возможность изменить его.

- Полагаться на мастер, который позволит вам изменить некоторые аспекты IDL-файла.

Теперь можно изменить в IDL-файл в файл исходного кода, используя атрибуты Visual C++ IDL. Во многих случаях атрибуты Visual C++ IDL имеют имя, совпадающее с именем атрибуты MIDL. Имя атрибута Visual C++ IDL и атрибута MIDL совпадают, означает, что Включение атрибута Visual C++ в файле исходного кода приведет к IDL-файл, содержащий ее тезка описании атрибута MIDL. Тем не менее атрибут Visual C++ IDL не предоставляют все возможности атрибута MIDL.

Если не используется с [атрибуты COM](../windows/com-attributes.md), атрибуты IDL позволяют определять интерфейсы. Если исходный код компилируется, атрибуты используются для определения созданного IDL-файла. При использовании с атрибуты COM в проект ATL, некоторые атрибуты IDL, такие как `coclass`, код, которые следует вставить в проект.

Обратите внимание, что [idl_quote](../windows/idl-quote.md) позволяет использовать MIDL конструкции, которые не поддерживаются в текущей версии Visual C++. Это и другие атрибуты, такие как [importlib](../windows/importlib.md) и [includelib](../windows/includelib-cpp.md) позволяют использовать существующие IDL-файлы в текущем проекте Visual C++.
