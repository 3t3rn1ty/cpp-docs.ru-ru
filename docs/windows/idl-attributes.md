# <a name="idl-attributes"></a>Атрибуты IDL
В большинстве случаев обслуживание IDL-файл предназначен было необходимо:  
  
-   Иметь представление о структуре и синтаксисе массива IDL-файл, чтобы иметь возможность изменить его.  
  
-   Использовать мастер, который дает возможность изменять некоторые аспекты IDL-файла.  
  
 Теперь можно изменить IDL-файл в файл исходного кода, используя атрибуты Visual C++ IDL. Во многих случаях атрибуты Visual C++ IDL имеют имя, совпадающее с именем атрибуты MIDL. Имя атрибута Visual C++ IDL и атрибут MIDL совпадают, означает, что Включение атрибута Visual C++ в файле исходного кода приведет к IDL-файл, содержащий его атрибут namesake MIDL. Однако атрибут Visual C++ IDL может не предоставлять все функциональные возможности атрибута MIDL.  
  
 Если не используется с [атрибуты COM](../windows/com-attributes.md), атрибуты IDL позволяют определять интерфейсы. При компиляции исходного кода, атрибуты используются для определения созданного IDL-файла. При использовании с атрибутами COM в проект ATL, некоторые атрибуты IDL, таких как **coclass**, вызывать код, чтобы вставить в проект.  
  
 Обратите внимание, что [idl_quote](../windows/idl-quote.md) позволяет использовать MIDL конструкции, которые не поддерживаются в текущей версии Visual C++. Это и другие атрибуты, такие как [importlib](../windows/importlib.md) и [includelib](../windows/includelib-cpp.md) позволяют использовать существующие IDL-файлы в текущем проекте Visual C++.
