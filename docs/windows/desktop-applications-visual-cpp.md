# Классические приложения Windows (Visual C++)
[!INCLUDE[vs2017banner](../assembler/inline/includes/vs2017banner.md)]

Вы можете создать классическое приложение Windows, если требуется классическое приложение неуправляемого кода с оконным пользовательским интерфейсом, которое может работать в версиях Windows от Windows XP до Windows 10 на настольных компьютерах Windows.  
  
 *Классическое приложение Windows* \(в старой литературе иногда называется приложением Win32\) — это общепринятый термин для приложения, использующего циклы непосредственной обработки сообщений Windows вместо использования платформы, например классов Microsoft Foundation Classes \(MFC\), библиотеки Active Template Library \(ATL\) или .NET Framework. Классическое приложение Windows в C\+\+ может использовать классы и функции среды выполнения C \(CRT\) и стандартной библиотеки шаблонов \(STL\), COM\-объекты и все открытые функции Windows, известные под общим названием "API Windows". Вводные сведения о классических приложениях Windows в C\+\+ см. в разделе [Учимся программировать для Windows в C\+\+](http://go.microsoft.com/fwlink/p/?LinkId=262281).  
  
 Классическое приложение Windows — это один из способов создания классического приложения неуправляемого кода для Windows. Другим способом является приложение MFC. MFC выбирается по умолчанию для приложений \(особенно приложений корпоративного типа\) с большим числом элементов управления ИП или пользовательских элементов управления. MFC предоставляет удобные вспомогательные классы для сериализации, манипуляций с текстом, печати и современных элементов интерфейса пользователя, таких как ленты. Эти классы недоступны для классического приложения Windows.
