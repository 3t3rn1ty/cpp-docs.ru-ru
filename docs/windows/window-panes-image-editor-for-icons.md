# <a name="window-panes-image-editor-for-icons"></a>Области окна (редактор изображений для значков)
Обычно окно редактора изображений показывает изображение в двух областях, разделенных линией. Одно представление является фактический размер, а другой увеличивается (коэффициент увеличения по умолчанию — 6). Автоматическое обновление представления в этих двух панелей: изменения, внесенные в одной области сразу же отображаются в другой. Области упрощают работать над увеличенное изображение, в котором можно различать отдельные точки и, в то же время результат работы на просмотр фактического размера изображения.  
  
 Слева использует как объем дискового пространства — необходимости (до половины окна изображения), чтобы отобразить представление масштабом 1:1 (по умолчанию) образа. На правой панели отображается увеличенное изображение (по умолчанию масштабе 6:1). Вы можете [изменение масштаба](../windows/changing-the-magnification-factor-image-editor-for-icons.md) в каждой области с помощью **Magnify** средство [панель инструментов редактора изображений](../windows/toolbar-image-editor-for-icons.md) или с помощью сочетания клавиш.  
  
 Можно увеличить область меньшего размера окна редактора изображений и использовать две области для отображения большого изображения. Щелкните в области, чтобы выбрать его.  
  
 Относительные размеры областей можно изменить, поместив указатель на разделитель и перемещение разделения влево или вправо. Разделитель можно перемещать на обеих сторонах, если необходимо работать с одной панелью.  
  
 Если области изображение увеличивается на коэффициент 4 или выше, вы можете [отображения пиксельной сетке](../windows/displaying-or-hiding-the-pixel-grid-image-editor-for-icons.md) , разделяет отдельные точки изображения.  
  
 Сведения о добавлении ресурсов в управляемые проекты см. в разделе [ресурсы в классических приложениях](/dotnet/framework/resources/index) в *руководства разработчика .NET Framework.* Сведения о вручную добавлять файлы ресурсов в управляемые проекты, осуществлять доступ к ресурсам, отображать статические ресурсы и присваивать строки ресурсов свойствам см. в разделе [Создание файлов ресурсов для приложений рабочего стола](/dotnet/framework/resources/creating-resource-files-for-desktop-apps).
