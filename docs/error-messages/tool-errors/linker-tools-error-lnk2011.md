# Ошибка средств компоновщика LNK2011
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

предкомпилированный объект не скомпонован; образ может не запуститься  
  
 При использовании предкомпилированных заголовков программе LINK необходимо, чтобы все объектные файлы, созданные с предкомпилированными заголовками, были скомпонованы.  При наличии исходного файла, используемого для создания предкомпилированных заголовков, которые применяются с другими исходными файлами, необходимо включить объектный файл, созданный вместе с предкомпилированным заголовком.  
  
 Например, если производится компиляция файла с именем STUB.cpp для создания предкомпилированного заголовка, который применяется для других исходных файлов, необходимо связать его с объектным файлом STUB.obj. В противном случае возникнет данная ошибка.  В приведенных ниже командных строках первая строка используется для создания предкомпилированного заголовка COMMON.pch, который применяется в файлах PROG1.cpp и PROG2.cpp во второй и третьей строке.  Файл STUB.cpp содержит только строки `#include` \(такие же строки `#include`, как в PROG1.cpp и PROG2.cpp\) и применяется только для создания предкомпилированных заголовков.  В последней строке STUB.obj должен быть скомпонован, чтобы не возникала ошибка LNK2011.  
  
```  
cl /c /Yccommon.h stub.cpp  
cl /c /Yucommon.h prog1.cpp  
cl /c /Yucommon.h prog2.cpp  
link /out:prog.exe stub.obj prog1.obj prog2.obj  
```
