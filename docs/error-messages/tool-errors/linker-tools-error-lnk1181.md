# Ошибка средств компоновщика LNK1181
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

не удается открыть входной файл "имяфайла"  
  
 Компоновщику не удалось найти файл `filename`, поскольку либо этот файл не существует, либо не найден путь.  
  
 Некоторые наиболее распространенные причины возникновения ошибки LNK1181:  
  
-   В строке компоновщика существует ссылка на файл `filename` как на дополнительную зависимость, но этот файл не существует.  
  
-   Не указан оператор **\/LIBPATH**, задающий каталог, в котором содержится файл `filename`.  
  
 Чтобы устранить эти проблемы, необходимо проверить, что все файлы, на которые есть ссылки в строке компоновщика, присутствуют в системе.  Также следует проверить наличие оператора **\/LIBPATH** для каждого каталога, содержащего нужный компоновщику файл.  
  
 Другая возможная причина ошибки LNK1181 состоит в том, что длинное имя файла, содержащее пробелы, не заключено в кавычки.  В этом случае компоновщик распознает только часть имени до первого пробела, после которой он предполагает расширение OBJ.  Чтобы разрешить эту ситуацию, следует заключить длинное имя файла \(путь и имя файла\) в кавычки.  
  
 Дополнительные сведения см. в разделе [LIB\-файлы как входные данные компоновщика](../../build/reference/dot-lib-files-as-linker-input.md).
