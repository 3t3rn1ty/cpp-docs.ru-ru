# <a name="c-runtime-error-r6030"></a>C R6030 ошибку во время выполнения
Библиотека CRT не инициализирована  
  
> [!NOTE]
>  Если это сообщение об ошибке возникает во время выполнения приложения, приложения была завершена из-за внутренней проблемы. Это наиболее часто возникает, некоторые программы для обеспечения безопасности, или в редких случаях ошибки в программе.  
>   
>  Для устранения этой ошибки попробуйте выполнить следующие действия:  
>   
>  -   Программное обеспечение безопасности может иметь определенные инструкции по устранению этой проблемы. Проверьте сайт поставщика программного обеспечения безопасности. Кроме того проверьте для обновленных версий программного обеспечения безопасности, или попробуйте программного обеспечения безопасности.  
> -   Используйте **программы и компоненты** или **программы и компоненты** страницы в **панели управления** восстановите или переустановите программу.  
> -   Проверьте **центра обновления Windows** в **панели управления** для обновлений программного обеспечения.  
> -   Проверьте наличие обновленной версии приложения. Обратитесь к поставщику приложения, если ошибка повторится.  
  
 **Сведения для программистов**  
  
 Эта ошибка возникает, если вы используете времени выполнения C (CRT), но код запуска CRT не был выполнен. Это возможно, эта ошибка возникает, если параметр компоновщика [/Entry](../../build/reference/entry-entry-point-symbol.md) используется для переопределения обычно начальный адрес, по умолчанию **mainCRTStartup**, **wmainCRTStartup** для консоли exe-файла, **случае** или **wWinMainCRTStartup** для EXE-файла Windows, или **_DllMainCRTStartup** для библиотеки DLL. Если во время запуска будет вызван один из перечисленных выше функций, среда выполнения языка C, не удалось инициализировать. При запуске вызываются эти функции обычно по умолчанию при связь с библиотекой времени выполнения C с использованием обычной **основной**, **wmain**, **WinMain**, или  **Функция DllMain** точки входа.  
  
 Можно также эта ошибка возникает, когда другая программа использует методы ввода кода для перехвата определенных вызовы библиотеки DLL. Некоторые программы продвинутых безопасности используют этот метод. В версиях Visual C++ до Visual Studio 2015 можно использовать статически связанные библиотеки CRT для устранения этой проблемы, но это не рекомендуется по соображениям обновлений безопасности и приложений. Исправление этой проблемы может потребовать действий пользователей.
