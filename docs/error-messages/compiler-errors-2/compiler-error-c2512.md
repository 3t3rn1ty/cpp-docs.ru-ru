# <a name="compiler-error-c2512"></a>Ошибка компилятора C2512

> "*идентификатор*": нет подходящего конструктора по умолчанию доступен  

Объект *конструктор по умолчанию*, конструктор, который не требует аргументов, не доступен для заданного класса, структуры или объединения. Компилятор предоставляет конструктор по умолчанию только в том случае, если нет пользовательских конструкторов предоставляются.

Если вы хотите разрешить создание класса, создаваемого без параметров (например, в качестве элементов в массиве) предоставляют конструктор, который принимает параметр, отличный от void, необходимо также указать конструктор по умолчанию. Конструктор по умолчанию может быть конструктором со значениями по умолчанию для всех параметров.

## <a name="example"></a>Пример

Является распространенной причиной ошибки C2512 при определении класса или структуры конструктор, который принимает аргументы, и попробуйте объявить экземпляр этого класса или структуры без аргументов. Например `struct B` ниже объявляет конструктор, который требует `char *` аргументом, но не тот, который не принимает аргументы. В `main`, экземпляр B объявлен, но аргумент не указан. Компилятор создает C2512, поскольку не удается найти конструктор по умолчанию для б.

```cpp
// C2512.cpp
// Compile with: cl /W4 c2512.cpp
// C2512 expected
struct B {
   B (char *) {}
   // Uncomment the following line to fix.
   // B() {}
};

int main() {
   B b;   // C2512 - This requires a default constructor
}
```

Эту проблему можно устранить путем определения конструктора по умолчанию для структуры или класса, такие как `B() {}`, или конструктор, где все аргументы имеют значения по умолчанию, таких как `B (char * = nullptr) {}`.
