# <a name="compiler-error-c2397"></a>C2397 ошибки компилятора
Преобразование «тип_1» в «тип2» требует сужающего преобразования  
  
 Неявные преобразования с сужением найден при использовании единообразная инициализация.  
  
 Языка C разрешает неявные сужающие преобразования в назначения и инициализации, и C++ следует масть, даже если Непредвиденная сужение является причиной многих ошибок кода. Чтобы сделать код безопаснее, стандарт языка C++ требует диагностическое сообщение при возникновении сужающее преобразование, в списке инициализации. В Visual C++ диагностические является C2397 ошибки компилятора при использовании единообразная инициализация начала синтаксис, поддерживаемый в Visual Studio 2015. Компилятор создает [C4838 Предупреждение компилятора (уровень 1)](../../error-messages/compiler-warnings/compiler-warning-level-1-c4838.md) при использовании списка или агрегатной инициализации синтаксис, поддерживаемый Visual Studio 2013.  
  
 Сужающее преобразование может быть нормально, если известно, что возможный диапазон значений может поместиться в целевом объекте. В этом случае известно более, чем компилятор. При внесении сужающее преобразование намеренно свои намерения явно объявить с помощью статическому типу. В противном случае это сообщение об ошибке почти всегда означает, что ошибки в коде. Вы можете исправить, убедившись, что объекты, которые можно инициализировать иметь типы, достаточное для обработки входных данных.  
  
 Следующий пример приводит к возникновению ошибки C2397 и показано, как исправить эту ошибку:  
  
```  
// C2397.cpp -- C++ narrowing conversion diagnostics  
// Compile by using: cl /EHsc C2397.cpp  
#include <vector>   
  
struct S1 {  
    int m1;  
    double m2, m3;  
};  
  
void function_C2397(double d1) {  
    char c1 { 127 };          // OK  
    char c2 { 513 };          // error C2397  
  
    std::vector<S1> vS1;  
    vS1.push_back({ d1, 2, 3 }); // error C2397  
  
    // Possible fix if you know d1 always fits in an int  
    vS1.push_back({ static_cast<int>(d1), 2, 3 });   
}  
```
