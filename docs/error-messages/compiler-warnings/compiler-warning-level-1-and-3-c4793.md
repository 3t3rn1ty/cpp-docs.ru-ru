# <a name="compiler-warning-level-1-and-3-c4793"></a>Предупреждение компилятора (уровни 1 и 3) C4793
«функция»: функция скомпилирована как машинный код: «причина»  
  
 Не удается скомпилировать *функция* в управляемом коде, даже если [/CLR](../../build/reference/clr-common-language-runtime-compilation.md) указан параметр компилятора. Вместо этого компилятор выдает предупреждение C4793 и пояснительное сообщение, а затем компилирует *функция* в машинный код. Продолжение сообщение содержит *Причина* текст, который объясняет, почему *функция* не может компилироваться в `MSIL`.  
  
 Это предупреждение уровня 1, при указании `/clr:pure` параметр компилятора.  **/CLR: pure** рекомендуется использовать параметр компилятора в Visual Studio 2015.  
  
 В следующей таблице перечислены все возможные причины возникновения этого сообщения.  
  
|Сообщение с описанием причины|Примечания|  
|--------------------|-------------|  
|Выровненные типы данных не поддерживаются в управляемом коде|CLR должен иметь возможность выделения данных, при необходимости, что не всегда возможно, если данные выравнивается объявления, такие как [__m128](../../cpp/m128.md) или [выравнивания](../../cpp/align-cpp.md).|  
|Функции, использующие «__ImageBase» не поддерживаются в управляемом коде|`__ImageBase`имеет специальный символ компоновщика обычно используется только в низкоуровневом неуправляемом коде для загрузки библиотеки DLL.|  
|varargs не поддерживается "/ clr" параметр компилятора|Собственный функциях не поддерживается вызов управляемых функций, содержащих [списки аргументов переменных](../../cpp/functions-with-variable-argument-lists-cpp.md) (varargs) из-за требований к размещению различных стека. Тем не менее если указать `/clr:pure` параметр компилятора, списки поддерживаются, поскольку сборка может содержать только управляемых функций переменное число аргументов. Дополнительные сведения см. в разделе [чистый и проверяемый код (C + +/ CLI)](../../dotnet/pure-and-verifiable-code-cpp-cli.md).|  
|64-разрядной среде CLR не поддерживает данные, объявленные с помощью модификатора __ptr32|Указатель должен быть по размеру неуправляемого указателя для текущей платформы. Дополнительные сведения см. в разделе [__ptr32, \__ptr64](../../cpp/ptr32-ptr64.md).|  
|32-разрядной среде CLR не поддерживает данные, объявленные с помощью модификатора __ptr64|Указатель должен быть по размеру неуправляемого указателя для текущей платформы. Дополнительные сведения см. в разделе [__ptr32, \__ptr64](../../cpp/ptr32-ptr64.md).|  
|Один или несколько встроенных функций не поддерживаются в управляемом коде|Имя встроенной функции не доступно в момент выдается сообщение. Тем не менее встроенная функция, вызывающая сообщение обычно представляет собой низкоуровневую машинную инструкцию.|  
|Встроенные машинные сборки (__asm) не поддерживаются в управляемом коде|[Встроенный код ассемблера](../../assembler/inline/asm.md) может содержать произвольное собственный код, который нельзя управлять.|  
|Преобразователь виртуальной функции не __clrcall необходимо компилировать как машинный код|Значение, отличное от[__clrcall](../../cpp/clrcall.md) преобразователь виртуальной функции необходимо использовать неуправляемый адрес.|  
|Функции с помощью «_setjmp» необходимо компилировать как машинный код|Среда CLR необходимо иметь возможность управлять выполнением программы. Тем не менее [setjmp](../../cpp/using-setjmp-longjmp.md) обходится выполнения программы, сохранение и низкоуровневой информации, например регистров и состояния выполнения.|  
  
## <a name="example"></a>Пример  
 Следующий пример приводит к возникновению ошибки C4793.  
  
```  
// C4793.cpp  
// compile with: /c /clr /W3   
// processor: x86  
int asmfunc(void) {   // C4793, compiled as unmanaged, native code  
   __asm {  
      mov eax, 0  
   }  
}  
```  
  
```Output  
warning C4793: 'asmfunc' : function is compiled as native code:  
        Inline native assembly ('__asm') is not supported in managed code  
```  
  
## <a name="example"></a>Пример  
 Следующий пример приводит к возникновению ошибки C4793.  
  
```  
// C4793_b.cpp  
// compile with: /c /clr /W3  
#include <setjmp.h>  
jmp_buf test_buf;  
  
void f() {  
   setjmp(test_buf);   // C4793 warning  
}  
```  
  
```Output  
warning C4793: 'f' : function is compiled as native code:  
        A function using '_setjmp' must be compiled as native  
```
