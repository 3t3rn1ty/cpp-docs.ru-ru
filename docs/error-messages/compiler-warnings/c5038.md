# <a name="compiler-warning-c5038"></a>C5038 Предупреждение компилятора

> элемент данных "*member1*«будет инициализирована после элемента данных»*member2*"  
> элемент данных "*член*«будет инициализирована после базовый класс»*base_class*"  

Члены класса инициализируются в порядке их объявления, а не порядке их отображения в списках инициализаторов. Это предупреждение означает, что порядок инициализации не является таким же, как порядок объявления членов данных или базовых классов. Это может привести к поведению не определено во время выполнения, если инициализация один элемент в списке зависит от инициализации член, объявленный в более поздней версии.

Это предупреждение новые возможности Visual Studio 2017 г. версия 15,3 и отключена по умолчанию. Используйте [/Wall](../../build/reference/compiler-option-warning-level.md) включить все предупреждения, отключенные по умолчанию или __/w__*n*__5038__ для включения C5038 как уровень *n* предупреждение. Дополнительные сведения см. в разделе [компилятора предупреждения, которые отключено по умолчанию](../../preprocessor/compiler-warnings-that-are-off-by-default.md). Сведения о том, как отключить предупреждения компилятора версии см. в разделе [предупреждения компилятора, версия компилятора](compiler-warnings-by-compiler-version.md).

## <a name="example"></a>Пример

В следующем примере вызывает Visual Studio 2017 г. версия 15,3 (с/Wall) «предупреждение C5038: данные-член «A::y» будет инициализирована после элемента данных «A::x»»:

```cpp
struct A
{
    A(int a) : y(a), x(y) {} // C5938 Initialized in reverse, y reused
    int x;
    int y;
};
```

Чтобы устранить эту проблему, расположите списка инициализаторов обязательно в порядке, соответствующем порядку декларации. Похожее предупреждение возникает, когда один инициализатор или оба ссылаются на члены базового класса.
