# <a name="compiler-warning-level-4-c4464"></a>Компилятора (уровень 4) предупреждение C4464

> **относительный путь включения содержит ".."**  

Объект `#include` директива имеет путь содержит ".." Описатель родительского каталога.

## <a name="remarks"></a>Примечания

Начиная с Visual Studio 2015 с обновлением 1, компилятор может обнаружить директива include, которая содержит ".." сегмент пути и проблема предупреждение, если параметр включен. Код, написанный таким образом, обычно предназначен для включения заголовков, находящихся за пределами проекта, путем неправильного использования относительных путей к проектам. Это создает риск, что компиляции программы мог быть путем включения другого исходного файла не планирует программист, или что эти относительные пути не было бы перенести в другие среды сборки. Несмотря на то, что нет конкретного не предупреждение для него, также рекомендуется не использовать каталоги включения сегмент родительский каталог пути для указания проекта.

Это предупреждение новые возможности Visual Studio 2015 с обновлением 1 и отключена по умолчанию. Используйте [/Wall](../../build/reference/compiler-option-warning-level.md) включить все предупреждения, отключенные по умолчанию или __/w__*n*__4464__ для включения C4464 как уровень *n* предупреждение. Дополнительные сведения см. в разделе [компилятора предупреждения, которые отключено по умолчанию](../../preprocessor/compiler-warnings-that-are-off-by-default.md). Сведения о том, как отключить предупреждения компилятора версии см. в разделе [предупреждения компилятора, версия компилятора](compiler-warnings-by-compiler-version.md).

## <a name="example"></a>Пример

Исходный код, содержащие ".." сегменты пути можно вызвать данное предупреждение, если **/Wall** параметра:

```cpp
#include "..\headers\C4426.h"  // emits warning C4464

// To fix this issue, specify only the header file name, and add
// the absolute path to 'headers\' to your project's include directories
#include "C4426.h"
```
