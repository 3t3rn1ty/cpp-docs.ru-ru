# <a name="compiler-warning-level-1-c4477"></a>Компилятор C4477 предупреждение (уровень 1)

> "*функция*": строка формата "*строка*«требует аргумент типа»*тип*", но аргумента с переменным числом аргументов *номер* имеет тип "*типа*"  

Компилятор обнаружил несоответствие между типом аргумента, необходимых для заполнителя в строке формата и тип аргумента. Правильное использование функций с переменным числом аргументов функции printf и scanf семейств требует, что аргументы типов, указанную в строке формата. Несоответствие, как правило, означает, что имеется ошибка в коде.

Сведения о аргументы, связанные с заполнителями строка семейства формата printf см [синтаксис описания формата: функции printf и wprintf](../../c-runtime-library/format-specification-syntax-printf-and-wprintf-functions.md). См. в документации для получения сведений о функции *функция*.

Это предупреждение впервые появился в Visual Studio 2015.

## <a name="example"></a>Пример

В этом примере показано два предупреждения C4477 для первой функции printf_s, если было найдено два аргумента неверного типа и также приводятся сведения по устранению проблемы.

```cpp
// C4477p.cpp
// compile with: cl /c /W4 C4477p.cpp
#include <stdio.h>

void print_func(int a, float f)
{
    printf_s("This takes int and double arguments %d and %f.\n", f, a); // C4477, expected int then double
    printf_s("This takes int and double arguments %d and %f.\n", a, f); // fix: types in correct order
}
```

В первом сообщении об ошибке, компилятор обнаруживает, что **двойные** аргумента при **int** ожидается. Объект **float** автоматически повышается до **двойные** в списке аргументов с переменным числом аргументов. Второе сообщение об ошибке показывает, что компилятор ожидал **двойные** аргумент, но **int** был предоставлен. Компилятор не может определить, что был переключен порядок аргументов. Необходимо проверить строку формата для определения порядка фактического аргумента и указать правильные типы аргументов.
