# <a name="styles-used-by-mfc"></a>Стили, используемые MFC  
  
Используйте следующие флаги стилей для указания внешнего вида окна или элемента управления и поведения при создании соответствующий объект MFC. В большинстве случаев эти стили заданы в `dwStyle` параметр класса `Create` функции.  
  
## <a name="button-styles"></a> Стили кнопок  
  
Применить стили к [класс CButton](../../mfc/reference/cbutton-class.md) проверьте объекты, такие как переключатели, поля и кнопок. Задать сочетание стилей в `dwStyle` параметр [CButton::Create](../../mfc/reference/cbutton-class.md#create). Дополнительные сведения о стилях кнопки в Windows см. в разделе [стили кнопок (Windows)](https://msdn.microsoft.com/library/windows/desktop/bb775951).  
  
### <a name="button-types"></a>Типы кнопок  
  
В следующей таблице перечислены типы кнопки. При необходимости можно выбрать одно из следующих. Если вы не укажете тип кнопки, значение по умолчанию — `BS_PUSHBUTTON`.  
  
|Тип|Описание|  
|----------|-----------------|  
|`BS_3STATE`|Создает кнопку флажок с тремя состояниями: `BST_CHECKED`, `BST_INDETERMINATE`, и `BST_UNCHECKED`. Нажав кнопку отправляет `BN_CLICKED` уведомления для окна-владельца, но не изменять состояние кнопки. По умолчанию связанный текст отображается справа от флажка. Для отображения текста в левом поле, используйте `BS_LEFTTEXT` или `BS_RIGHTBUTTON` стиль.|  
|`BS_AUTO3STATE`|Создает кнопку флажок с тремя состояниями: `BST_CHECKED`, `BST_INDETERMINATE`, и `BST_UNCHECKED`. Нажав кнопку отправляет `BN_CLICKED` уведомления окну-владельцу и изменяет состояние кнопки. Состояния кнопок цикла порядка `BST_CHECKED`, `BST_INDETERMINATE`, и `BST_UNCHECKED`. По умолчанию связанный текст отображается справа от флажка. Для отображения текста в левом поле, используйте `BS_LEFTTEXT` или `BS_RIGHTBUTTON` стиль.|  
|`BS_AUTOCHECKBOX`|Создает кнопку флажок в двух состояниях: `BST_CHECKED` и `BST_UNCHECKED`. Нажав кнопку отправляет `BN_CLICKED` уведомления окну-владельцу и изменяет состояние кнопки. По умолчанию связанный текст отображается справа от флажка. Для отображения текста в левом поле, используйте `BS_LEFTTEXT` или `BS_RIGHTBUTTON` стиль.|  
|`BS_AUTORADIOBUTTON`|Создает переключатель с двумя состояниями: `BST_CHECKED` и `BST_UNCHECKED`. Переключатели обычно используются в группах с каждой группой, имеющих более одного отмечена флажком одновременно. Нажав кнопку отправляет `BN_CLICKED` уведомление, чтобы окно-владелец, задает состояние пользователь щелкнул переключатель, чтобы `BST_CHECKED`и задает состояния всех остальных переключателей в группе переключателей для `BST_UNCHECKED`. По умолчанию связанный текст отображается справа от переключателя. Чтобы отобразить текст слева от "переключатель", используйте `BS_LEFTTEXT` или `BS_RIGHTBUTTON` стиль.|  
|`BS_CHECKBOX`|Создает кнопку флажок в двух состояниях: `BST_CHECKED` и `BST_UNCHECKED`. Нажав кнопку отправляет `BN_CLICKED` уведомления для окна-владельца, но не изменять состояние кнопки. По умолчанию связанный текст отображается справа от флажка. Для отображения текста в левом поле, используйте `BS_LEFTTEXT` или `BS_RIGHTBUTTON` стиль.|  
|`BS_COMMANDLINK`|Создает ссылку кнопки. Командная кнопка ссылки является Командная кнопка, относящиеся к [!INCLUDE[windowsver](../../build/reference/includes/windowsver_md.md)] , отображается зеленая стрелка слева от основного текста и примечание ниже основного текста. Можно задать текст заметки с помощью [CButton::SetNote](../../mfc/reference/cbutton-class.md#setnote).|  
|`BS_DEFCOMMANDLINK`|Создает ссылку кнопки. Командная кнопка ссылки является Командная кнопка, относящиеся к [!INCLUDE[windowsver](../../build/reference/includes/windowsver_md.md)] , отображается зеленая стрелка слева от основного текста и примечание ниже основного текста. Можно задать текст заметки с помощью [CButton::SetNote](../../mfc/reference/cbutton-class.md#setnote). Если кнопка находится в диалоговое окно, нажав клавишу ввод ключа отправляет `BN_CLICKED` уведомлений диалоговым окном даже в том случае, если кнопка не имеет фокус ввода.|  
|`BS_DEFPUSHBUTTON`|Создание кнопки с интенсивной черной рамкой. Если кнопка находится в диалоговое окно, нажав клавишу ввод ключа отправляет `BN_CLICKED` уведомлений диалоговым окном даже в том случае, если кнопка не имеет фокус ввода.|  
|`BS_DEFSPLITBUTTON`|Создает элемент разворачивающейся кнопки. Разворачивающаяся кнопка — Командная кнопка, относящиеся к [!INCLUDE[windowsver](../../build/reference/includes/windowsver_md.md)] , содержащий рядом раскрывающуюся стрелку кнопки. При нажатии кнопки по умолчанию команда выполняется. Если щелкнуть стрелку раскрывающегося списка, появится меню дополнительных команд. Если Разворачивающаяся кнопка находится в диалоговое окно, нажав клавишу ввод ключа отправляет `BN_CLICKED` уведомлений диалоговым окном даже в том случае, если кнопка не имеет фокуса ввода|  
|`BS_GROUPBOX`|Создает прямоугольник, в котором могут быть сгруппированы другие кнопки. Текст, связанный с этим стилем отображается в верхнего левого угла прямоугольника.|  
|`BS_OWNERDRAW`|Создает кнопку владельцем. Платформа вызывает `DrawItem` метод при изменении внешнего вида кнопки был изменен. Необходимо задать этот стиль, при использовании `CBitmapButton` класса.|  
|`BS_PUSHBUTTON`|Создает кнопку, отправляет `BN_CLICKED` уведомления для окна-владельца, когда пользователь нажимает кнопку.|  
|`BS_RADIOBUTTON`|Создает переключатель с двумя состояниями: `BST_CHECKED` и `BST_UNCHECKED`. Переключатели обычно используются в группах с каждой группой, имеющих более одного отмечена флажком одновременно. Чтобы отобразить текст слева от "переключатель", используйте `BS_LEFTTEXT` или `BS_RIGHTBUTTON` стиль.|

### <a name="radio-button-and-check-box-styles"></a>Стили и флажков радио  
  
В следующей таблице перечислены стили, которые применяются к переключатели и флажки. Эти стили игнорируются все другие типы кнопок. При необходимости можно выбрать один или несколько из следующих.  
  
|Стиль|Описание|  
|-----------|-----------------|  
|`BS_LEFTTEXT`|В сочетании со стилем кнопку или флажок переключатель, текст отображается слева от "переключатель" или "флажок".|  
|`BS_RIGHTBUTTON`|В сочетании со стилем кнопку или флажок переключатель, текст отображается слева от "переключатель" или "флажок". Этот стиль идентична `BS_LEFTTEXT` стиля.|  
|`BS_PUSHLIKE`|Делает флажок или переключатель выглядят и работают так, как кнопка команды. Кнопка выглядит нажатой, когда он находится в состоянии `BST_CHECKED`, нажатии и затемнены, когда он находится в состоянии `BST_INDETERMINATE`и снимаются, когда он находится в состоянии `BST_UNCHECKED`.|  
  
### <a name="button-text-alignment-styles"></a>Стили выравнивание текста кнопки  
  
В следующей таблице перечислены параметры выравнивания текста по горизонтали и вертикали. При необходимости можно выбрать одно из следующих.  
  
|Стиль|Описание|  
|-----------|-----------------|  
|`BS_LEFT`|По левому краю в прямоугольнике кнопки. Тем не менее, если кнопке представляет собой флажок или переключатель, который не имеет `BS_RIGHTBUTTON` стиля, остается текст выровнены по правой части флажок или переключатель.|  
|`BS_RIGHT`|Текст в прямоугольнике кнопку выравнивается вправо. Тем не менее, если кнопке представляет собой флажок или переключатель, который не имеет `BS_RIGHTBUTTON` стиля, текст выравнивается по правой справа от флажка или переключателя.|  
|`BS_CENTER`|Выравнивание текста по горизонтали в прямоугольнике кнопки.|  
|`BS_TOP`|Текст выравнивается по верхней стороны прямоугольника кнопки.|  
|`BS_BOTTOM`|Заключает текст в нижней части кнопки прямоугольника.|  
|`BS_VCENTER`|Центрирование текста по вертикали в прямоугольнике кнопки.|  
  
### <a name="button-content-options"></a>Параметры содержимого кнопки  
  
В следующей таблице перечислены параметры, которые указывают, что отображается на кнопке. Типы кнопок, отображающие текст только игнорировать эти стили. При необходимости можно выбрать одно из следующих.  
  
|Стиль|Описание|

|`BS_ICON`|Указывает, что кнопка отображается значок.|  
|`BS_TEXT`|Указывает, что на кнопке отображается текст.|  
  
### <a name="other-button-options"></a>Другие параметры кнопки  
  
В следующей таблице перечислены дополнительные параметры, которые можно использовать с любым типом кнопки. При необходимости можно выбрать один или несколько из следующих.  
  
|Стиль|Описание|  
|-----------|-----------------|  
|`BS_FLAT`|Указывает, что кнопки двухмерной и не рисуется с помощью заливки по умолчанию для создания трехмерного изображения.|  
|`BS_MULTILINE`|Если текстовая строка слишком длинна в одной строке в прямоугольнике кнопка заключает в оболочку на кнопке отображается надпись в несколько строк.|  
|`BS_NOTIFY`|Активирует кнопку Отправить `BN_DBLCLK`, `BN_KILLFOCUS`, и `BN_SETFOCUS` сообщения уведомления своему родительскому окну. Обратите внимание, что кнопок отправки `BN_CLICKED` уведомление, независимо от того, указана ли этот стиль.|  
  
## <a name="combo-box-styles"></a>  Стили полей со списками  
  
В MFC доступны перечисленные далее стили полей со списками. Дополнительные сведения о стилях поле со списком в Windows см. в разделе [стили полей со списком (Windows)](https://msdn.microsoft.com/library/windows/desktop/bb775796).  
  
|Стиль|Описание|  
|-----------|-----------------|  
|`CBS_AUTOHSCROLL`|Автоматическая прокрутка текста в элементе управления вправо при вводе очередного знака в конец строки. Если этот стиль не задан, допускается только текст, который умещается в прямоугольной области.|  
|`CBS_DISABLENOSCROLL`|Отображается отключенная вертикальная полоса прокрутки при списке содержится недостаточно элементов для прокрутки. Если этот стиль не задан, полоса прокрутки будет скрыта при недостаточном количестве элементов в списке.|

|`CBS_OWNERDRAWVARIABLE`|Владелец списка отвечает за рисование его содержимого; элементы в списке имеют разную высоту.|  
|`CBS_SIMPLE`|Окно списка отображается всегда. Текущее выделение в списке отображается в элементе управления "Поле ввода".|  
|`CBS_SORT`|Автоматическая сортировка строк, введенных в поле со списком.|  
|`CBS_UPPERCASE`|Преобразует в верхний регистр весь текст в поле выбора и в списке.|

|`SBS_RIGHTALIGN`|При использовании `SBS_VERT` стиля. Правый край полосы прокрутки выравнивается с правого края прямоугольника, заданного в `Create` функции-члена. Полоса прокрутки имеет ширину по умолчанию для системы полосы прокрутки.|  
|`SBS_SIZEBOX`|Задает размер поля. Если ни одна из `SBS_SIZEBOXBOTTOMRIGHTALIGN` , ни `SBS_SIZEBOXTOPLEFTALIGN` указанного стиля, поле «размер» имеет высоту, ширину и положение, предоставленное `Create` функции-члена.|  
|`SBS_SIZEBOXBOTTOMRIGHTALIGN`|При использовании `SBS_SIZEBOX` стиля. В нижнем правом углу поле размер выравнивается нижнего правого угла прямоугольника, указанного в `Create` функции-члена. Поле «размер» имеет размер по умолчанию для системы размер поля.|  
|`SBS_SIZEBOXTOPLEFTALIGN`|При использовании `SBS_SIZEBOX` стиля. Верхнего левого угла прямоугольника размер выравнивается верхнего левого угла прямоугольника, указанного в `Create` функции-члена. Поле «размер» имеет размер по умолчанию для системы размер поля.|  
|`SBS_SIZEGRIP`|То же, что `SBS_SIZEBOX`, но приподнятую границу.|  
|`SBS_TOPALIGN`|При использовании `SBS_HORZ` стиля. В верхней части полосы прокрутки выравнивается верхний край прямоугольника, заданного в `Create` функции-члена. Полоса прокрутки имеет высоту по умолчанию для системы полосы прокрутки.|  
|`SBS_VERT`|Задает вертикальную полосу прокрутки. Если ни одна из `SBS_RIGHTALIGN` , ни `SBS_LEFTALIGN` задан, полоса прокрутки имеет высоту, ширину и положение, предоставленное `Create` функции-члена.|  
  
## <a name="static-styles"></a> Статические стили  
  
Статические стили применяются к [CStatic класс](../../mfc/reference/cstatic-class.md) объектов. Задать сочетание стилей в `dwStyle` параметр [CStatic::Create](../../mfc/reference/cstatic-class.md#create). Дополнительные сведения о стилях статический элемент управления в Windows см. в разделе [статические стили элемента управления (Windows)](http://msdn.microsoft.com/library/windows/desktop/bb760773).  
  
|Стиль|Описание|  
|-----------|-----------------|  
|`SS_BITMAP`|Указывает, что является растровое изображение для отображения в статический элемент управления. Данный текст — это имя растрового изображения (не является именем файла), определенный в другом месте в файле ресурсов. Стиль игнорирует параметры nWidth и nHeight; элемент управления автоматически изменяет свой размер для размещения растрового изображения.|  
|`SS_BLACKFRAME`|Определяет окно фрейма, нарисованных при помощи совпадает с цветом рамки окна. Значением по умолчанию является Black.|  
|`SS_BLACKRECT`|Задает прямоугольник, закрашенный цвет, используемый для рисования рамки окна. Значением по умолчанию является Black.|  
|`SS_CENTER`|Обозначает простой прямоугольник и отображает заданный текст по центру в прямоугольнике. Перед отображением форматирования текста. Слова, которые распространится после конца строки, автоматически переносятся в начало следующей строки по центру.|  
|`SS_CENTERIMAGE`|Указывает, что в случае, если точечный рисунок или значок меньше клиентской области статического элемента управления, остальная часть клиентской области заполняется цвет пикселя в левом верхнем углу точечный рисунок или значок. Если статический элемент управления содержит одну строку текста, текст выравнивается по центру по вертикали в клиентской области элемента управления.|  
|`SS_ENDELLIPSIS`|или `SS_PATHELLIPSIS` заменяет часть заданной строки эллипсы, при необходимости, чтобы результат соответствовал размеру в заданном прямоугольнике. <br /><br />Можно указать `SS_END_ELLIPSIS` для замены символов в конце строки или `SS_PATHELLIPSIS` для замены символов в середине строки. Если строка содержит обратную косую черту (\\) символов, `SS_PATHELLIPSIS` сохраняет максимально возможную часть текста после последней обратной косой черты можно.|  
|`SS_ENHMETAFILE`|Указывает, что расширенный метафайл будет отображаться в статический элемент управления. Данный текст является имя метафайл. Расширенный метафайл статический элемент управления имеет фиксированный размер. метафайл масштабируется по размерам клиентской области статического элемента управления.|  
|`SS_ETCHEDFRAME`|Рисует кадр статический элемент управления с помощью `EDGE_ETCHED` ребро стиля.|  
|`SS_ETCHEDHORZ`|Рисует верхней и нижней границ статический элемент управления с помощью `EDGE_ETCHED` ребро стиля.|  
|`SS_ETCHEDVERT`|Рисует левого и правого краев статического элемента управления с использованием стиля границы EDGE_ETCHED.|  
|`SS_GRAYFRAME`|Определяет окно фрейма, созданного с тем же цветом фона экрана (для настольных компьютеров). По умолчанию задано значение Gray.|  
|`SS_GRAYRECT`|Задает прямоугольник, закрашенный цвет, используемый для заливки фона экрана. По умолчанию задано значение Gray.|  
|`SS_ICON`|Задает значок отображается в диалоговом окне. Данный текст — это имя значка (не является именем файла), определенный в другом месте в файле ресурсов. `nWidth` И `nHeight` параметры учитываются; значок автоматически изменяет свой размер.|  
|`SS_LEFT`|Обозначает простой прямоугольник и отображает заданный текст левого flush в прямоугольнике. Перед отображением форматирования текста. Слова, которые распространится после конца строки, автоматически переносятся в начало следующей строки Сброс влево.|  
|`SS_LEFTNOWORDWRAP`|Обозначает простой прямоугольник и отображает заданный текст левого flush в прямоугольнике. Вкладки, расширяются таким образом, но слова не заключены в оболочку. Текст, который выходит за пределы строки обрезается.|  
|`SS_NOPREFIX`|Если этот стиль не задан, Windows будет интерпретировать любой знаков амперсанда (&) в тексте элемента управления будут символы префикса сочетаний клавиш. В этом случае амперсанд удаляется и подчеркнуты следующий символ в строке. Если статический элемент управления должен содержать текст которой эта функция не нужен, `SS_NOPREFIX` могут быть добавлены. Этот стиль статического управления могут быть включены с любой из определенных статические элементы управления. Можно объединять `SS_NOPREFIX` с другим стилем с помощью побитового оператора OR. Наиболее часто используется при имена файлов или других строк, которые могут содержать знак амперсанда необходимо отображать в статический элемент управления в диалоговом окне.|  
|`SS_NOTIFY`|Родительское окно отправляет `STN_CLICKED`, `STN_DBLCLK`, `STN_DISABLE`, и `STN_ENABLE` сообщения уведомления, когда пользователь нажимает кнопку или дважды щелкает элемент управления.|  
|`SS_OWNERDRAW`|Указывает, что владельцем статический элемент управления отвечает за рисование элемента управления. Получает окно-владелец `WM_DRAWITEM` сообщение каждый раз, когда нужно отрисовать элемента управления.|  
|`SS_REALSIZEIMAGE`|Предотвращает статический элемент управления значка или точечного рисунка (то есть статические элементы управления, имеющие `SS_ICON` или `SS_BITMAP` стиль) изменения размеров при загрузке или рисования. Если значок или растровое изображение больше, чем область назначения, изображение обрезается.|  
|`SS_RIGHT`|Обозначает простой прямоугольник и отображает заданный текст правом записи в прямоугольнике. Перед отображением форматирования текста. Слова, которые распространится после конца строки, автоматически переносятся в начало следующей строки правом записи.|  
|`SS_RIGHTJUST`|Указывает, что правый нижний угол статический элемент управления с `SS_BITMAP` или `SS_ICON` стиль фиксируется оставаться при изменении размера элемента управления. Только верхней и левой сторон подгоняются в соответствии нового точечный рисунок или значок.|  
|`SS_SIMPLE`|Обозначает простой прямоугольник и отображается одна строка текста flush левого в прямоугольнике. Строка текста не может сокращено или изменен каким-либо образом. (Родительского элемента управления или диалогового окна поле не должно обрабатывать `WM_CTLCOLOR` сообщений.)|  
|`SS_SUNKEN`|Рисует утопленная линия границы вокруг статический элемент управления.|  
|`SS_USERITEM`|Указывает определяемый пользователем элемент.|  
|`SS_WHITEFRAME`|Определяет окно фрейма, нарисованных при помощи совпадает с цветом фона окна. По умолчанию цвет белый.|  
|`SS_WHITERECT`|Задает прямоугольник, закрашенный цвет, используемый для заливки фона окна. По умолчанию цвет белый.|  
|`SS_WORDELLIPSIS`|Усекает текст, который не умещается и добавляется многоточие.|  
  
## <a name="window-styles"></a> Стили окна  
  
Стили окна применяются к [класс CWnd](../../mfc/reference/cwnd-class.md) объектов. Задать сочетание стилей в `dwStyle` параметр [CWnd::Create](../../mfc/reference/cwnd-class.md#create) или [CWnd::CreateEx](../../mfc/reference/cwnd-class.md#createex). Дополнительные сведения о стилях окна в Windows см. в разделе [стили окна (Windows)](https://msdn.microsoft.com/library/windows/desktop/ms632600).  
  
|Стиль|Описание|  
|-----------|-----------------|  
|`WS_BORDER`|Создает границу окна.|  
|`WS_CAPTION`|Создает окно, которое имеет заголовок (подразумевает `WS_BORDER` стиль). Нельзя использовать с `WS_DLGFRAME` стиля.|  
|`WS_CHILD`|Создает дочернее окно. Нельзя использовать с `WS_POPUP` стиля.|  
|`WS_CHILDWINDOW`|То же, что `WS_CHILD` стиля.|  
|`WS_CLIPCHILDREN`|Исключает область, занимаемую дочерних окон при рисовании в родительском окне. Используется при создании родительского окна.|  
|`WS_CLIPSIBLINGS`|Отсекает дочерних окон друг относительно друга; то есть, в том случае, когда определенного дочернее окно получает отправку сообщения `WS_CLIPSIBLINGS` стиль отсекает всех остальных перекрывающихся дочерних окон за пределы области дочернее окно обновления. (Если `WS_CLIPSIBLINGS` не предоставляется, и дочерние окна перекрывают друг друга, при рисовании в клиентской области дочернего окна, возможно для рисования в клиентской области соседних дочернего окна.) Для использования с `WS_CHILD` только стиля.|  
|`WS_DISABLED`|Создает окно, которое изначально отключены.|  
|`WS_DLGFRAME`|Создает окно с двойной границей, но без заголовка.|  
|`WS_GROUP`|Указывает первый элемент управления из группы элементов управления, в которых пользователь может перемещать из одного элемента управления к другому с помощью клавиш со стрелками. Все элементы управления, имеющий `WS_GROUP` стиль `FALSE` после первого элемента управления принадлежать одной группе. Следующий элемент управления с `WS_GROUP` стиль начинается следующей группы (то есть одна группа заканчивается до начала следующего).|  
|`WS_HSCROLL`|Создает окно с горизонтальной полосы прокрутки.|  
|`WS_ICONIC`|Создает окно, которое изначально свернуто. То же, что `WS_MINIMIZE` стиля.|  
|`WS_MAXIMIZE`|Создает окно максимальный размер.|  
|`WS_MAXIMIZEBOX`|Создает окно, содержащее кнопку развертывания.|  
|`WS_MINIMIZE`|Создает окно, которое изначально свернуто. Для использования с `WS_OVERLAPPED` только стиля.|  
|`WS_MINIMIZEBOX`|Создает окно, содержащее кнопку свертывания.|  
|`WS_OVERLAPPED`|Создает перекрывающееся окно. Перекрывающееся окно обычно имеет заголовок и границы.|  
|`WS_OVERLAPPEDWINDOW`|Создает перекрывающееся окно с `WS_OVERLAPPED`, `WS_CAPTION`, `WS_SYSMENU`, `WS_THICKFRAME`, `WS_MINIMIZEBOX`, и `WS_MAXIMIZEBOX` стили.|  
|`WS_POPUP`|Создание всплывающего окна. Нельзя использовать с `WS_CHILD` стиля.|  
|`WS_POPUPWINDOW`|Создающий всплывающее окно с `WS_BORDER`, `WS_POPUP`, и `WS_SYSMENU` стили. `WS_CAPTION` Стиля должны быть объединены с `WS_POPUPWINDOW` стиль для отображения элемента управления меню.|  
|`WS_SIZEBOX`|Создает окно, которое имеет границы для изменения размера. То же, что `WS_THICKFRAME` стиля.|  
|`WS_SYSMENU`|Создает окно, которое имеется поле меню управления в своем заголовке. Используется только для windows с помощью заголовков.|  
|`WS_TABSTOP`|Задает один из любого числа элементов управления, через которые можно перемещаться с помощью клавиши TAB. Пользователь перемещается клавишей TAB к следующему элементу управления, определяемое `WS_TABSTOP` стиля.|  
|`WS_THICKFRAME`|Создает окно с утолщенной границей фрейма, можно использовать для изменения размеров окна.|  
|`WS_TILED`|Создает перекрывающееся окно. Перекрывающееся окно имеет заголовок и границы. То же, что `WS_OVERLAPPED` стиля.|  
|`WS_TILEDWINDOW`|Создает перекрывающееся окно с `WS_OVERLAPPED`, `WS_CAPTION`, `WS_SYSMENU`, `WS_THICKFRAME`, `WS_MINIMIZEBOX`, и `WS_MAXIMIZEBOX` стили. То же, что `WS_OVERLAPPEDWINDOW` стиля.|  
|`WS_VISIBLE`|Создает окно, которое изначально является видимым.|  
|`WS_VSCROLL`|Создает окно, которое имеет вертикальную полосу прокрутки.|  
  
## <a name="extended-window-styles"></a> Расширенные стили окна  
  
Расширенные стили окна применяются к [класс CWnd](../../mfc/reference/cwnd-class.md) объектов. Задать сочетание стилей в `dwExStyle` параметр [CWnd::CreateEx](../../mfc/reference/cwnd-class.md#createex). Дополнительные сведения о расширенные стили окна в Windows см. в разделе [расширенные стили окна (Windows)](https://msdn.microsoft.com/library/windows/desktop/ff700543).  
  
|Стиль|Описание|  
|-----------|-----------------|  
|`WS_EX_ACCEPTFILES`|Указывает, что окно, созданных с помощью этого стиля принимает и перетащите файлы.|  
|`WS_EX_APPWINDOW`|Принудительно окна верхнего уровня на панели задач, если окно является видимым.|  
|`WS_EX_CLIENTEDGE`|Указывает, что окно имеет объемный вид — то есть граница утопленную границу.|  
|`WS_EX_CONTEXTHELP`|Включает вопросительный знак в строке заголовка окна. Когда пользователь щелкает вопросительный знак, курсор изменяется с вопросительным знаком с указателем. Если пользователь щелкает дочернее окно, это окно получает `WM_HELP` сообщения.|  
|`WS_EX_CONTROLPARENT`|Позволяет пользователю перемещаться между окнами дочернего окна с помощью клавиши TAB.|  
|`WS_EX_DLGMODALFRAME`|Назначает окно с двойной границей, который (при необходимости) может быть создан с использованием заголовка при указании `WS_CAPTION` стиля флаг в `dwStyle` параметра.|  
|`WS_EX_LAYERED`|Окно [многоуровневых окон](http://msdn.microsoft.com/library/ms632599.aspx#layered). Этот стиль не может использоваться, если окно [стиль класса](http://msdn.microsoft.com/library/ms633574.aspx#class_styles) любого `CS_OWNDC` или `CS_CLASSDC`. Тем не менее [!INCLUDE[win8_first](../../mfc/reference/includes/win8_first_md.md)] поддерживает `WS_EX_LAYERED` стиля для дочерних окон, где предыдущих версий Windows поддерживают его только для окон верхнего уровня.|  
|`WS_EX_LEFT`|Предоставляет окна универсальные свойства по левому краю. Это значение по умолчанию.|  
|`WS_EX_LEFTSCROLLBAR`|Помещает вертикальная полоса прокрутки слева от клиентской области.|  
|`WS_EX_LTRREADING`|Отображает текст окна, с помощью слева направо считывания свойств заказа. Это значение по умолчанию.|  
|`WS_EX_MDICHILD`|Создает дочернего окна MDI.|  
|`WS_EX_NOPARENTNOTIFY`|Указывает, что дочернее окно, созданных с помощью этот стиль не будет отправлять `WM_PARENTNOTIFY` сообщение своему родительскому окну при создании или уничтожении дочернего окна.|  
|`WS_EX_OVERLAPPEDWINDOW`|Объединяет `WS_EX_CLIENTEDGE` и `WS_EX_WINDOWEDGE` стили|  
|`WS_EX_PALETTEWINDOW`|Объединяет `WS_EX_WINDOWEDGE` и `WS_EX_TOPMOST` стили.|  
|`WS_EX_RIGHT`|Предоставляет окна универсальные свойства, по правому краю. Это определяется класс окна.|  
|`WS_EX_RIGHTSCROLLBAR`|Помещает вертикальной полосы прокрутки в правой части клиентской области (при его наличии). Это значение по умолчанию.|  
|`WS_EX_RTLREADING`|Отображает текст окна, с помощью чтения справа налево порядок свойства.|  
|`WS_EX_STATICEDGE`|Создает окно со стилем трехмерная граница предназначен для использования для элементов, которые не принимают пользовательский ввод.|  
|`WS_EX_TOOLWINDOW`|Создает окно инструментов, то есть окна, предназначен для использования в качестве перемещаемой панелью инструментов. Окно инструментов имеет заголовок, короче обычного заголовка и выводится более мелким шрифтом. Окно инструментов не отображается на панели задач или в окне, которое появляется при нажатии клавиши ALT + TAB.|  
|`WS_EX_TOPMOST`|Указывает, что окно, созданных с помощью этого стиля должны находиться над всеми nontopmost windows и оставаться над ними даже в том случае, когда окно перестает быть активным. Приложение может использовать `SetWindowPos` функции-члена для добавления или удаления этого атрибута.|  
|`WS_EX_TRANSPARENT`|Указывает, что окно, созданных с помощью этого стиля является прозрачным. То есть любой версии windows, которые находятся под окна не закрытых окном. Получает окно, созданных с помощью этого стиля `WM_PAINT` сообщений только после обновления все окна того же уровня под ним.|  
|`WS_EX_WINDOWEDGE`|Указывает, что окно имеет приподнятую границу границы.|
