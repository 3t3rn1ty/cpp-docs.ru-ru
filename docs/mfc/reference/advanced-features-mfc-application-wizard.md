# Страница &quot;Дополнительные функции&quot; мастера приложений MFC
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

В этом разделе представлены параметры, относящиеся к дополнительным возможностям приложения, таким как справка, поддержка печати и т. д.  С помощью данных параметров можно включить поддержку дополнительных возможностей, описанных ниже.  
  
 **Контекстная справка \(HTML\)**  
 Создание набора справочных файлов для контекстной справки, доступ к которой можно получить, нажав клавишу F1, выбрав пункт меню "Справка" или нажав кнопку **Справка** в диалоговом окне.  Для поддержки справки требуется компилятор справки.  Если компилятор справки отсутствует, его можно установить, заново запустив программу установки.  
  
 Подробнее: [Справка HTML. Контекстная справка для программ](../../mfc/html-help-context-sensitive-help-for-your-programs.md) и [Файлы справки \(справка HTML\)](../../ide/help-files-html-help.md).  
  
 **печать и предварительный просмотр печати.**  
 Создает код для обработки команд печати, параметров печати и предварительного просмотра посредством вызова функций\-членов из класса [CView Class](../Topic/CView%20Class.md) библиотеки MFC.  Мастер также добавляет в меню приложения команды для этих функций.  Поддержка печати доступна только для приложений, у которых на странице  мастера приложения MFC [Тип приложения, мастер приложений MFC](../Topic/Application%20Type,%20MFC%20Application%20Wizard.md) выбран параметр **Поддержка архитектуры Document\/View**.  По умолчанию приложения, использующие архитектуру "документ\-представление", поддерживают печать.  
  
 **Автоматизация**  
 Задает, может ли приложение управлять объектами, реализованными в другом приложении, или предоставляет ли клиентам автоматизации доступ к приложению.  
  
 **Элементы управления ActiveX**  
 Указывает, включена ли поддержка элементов управления ActiveX \(выбрано по умолчанию\).  Если этот параметр не выбран, но затем потребовалось вставить в проект элементы управления ActiveX, необходимо добавить вызов [AfxEnableControlContainer](../Topic/AfxEnableControlContainer.md) в функцию\-член [CWinApp::InitInstance](../Topic/CWinApp::InitInstance.md) приложения.  
  
 **Интерфейс MAPI \(Messaging API\)**  
 Указывает, может ли приложение создавать, передавать и хранить почтовые сообщения, а также управлять ими.  
  
 **Сокеты Windows**  
 Поддержка служб Windows Sockets, с помощью которых можно создавать приложения, обменивающиеся данными по сетям TCP\/IP.  
  
 **Библиотека Active accessibility**  
 Добавление поддержки интерфейса [IAccessible](http://msdn.microsoft.com/library/windows/desktop/dd318466) в классы, производные от [CWnd](../Topic/CWnd%20Class.md), что позволяет настраивать пользовательский интерфейс для улучшения взаимодействия с клиентами специальных возможностей.  
  
 **Манифест стандартных элементов управления**  
 По умолчанию включено.  Создание манифеста приложения для включения новой библиотеки DLL стандартных элементов управления, поставляемой с Microsoft Windows XP или операционными системами последующих поколений.  
  
 Версия 6 библиотеки DLL стандартных элементов управления не поддерживает автоматическое обновление предыдущих версий стандартных элементов управления, используемых существующим приложением.  Чтобы использовать версию 6 библиотеки DLL стандартных элементов управления, необходимо создать манифест приложения, указывающий приложению загрузить новую библиотеку DLL.  Эта библиотека DLL стандартных элементов управления также поддерживает темы Windows XP.  
  
 Манифест приложения может также задавать другие библиотеки DLL и версии библиотек, необходимые приложению.  Дополнительные сведения о манифестах приложения см. в пакете [Изолированные приложения и параллельные сборки](http://msdn.microsoft.com/library/dd408052) в разделе [!INCLUDE[winsdkshort](../../atl/reference/includes/winsdkshort_md.md)].  
  
 **Поддержка диспетчера перезагрузки**  
 Добавляет поддержку [диспетчера перезапуска Windows](http://msdn.microsoft.com/library/windows/desktop/aa373680\(v=vs.85\).aspx).  В этом видео показано, как использовать диспетчер перезапуска MFC: [Практические советы. Использование нового диспетчера перезапуска.](http://msdn.microsoft.com/vstudio/ee886407).  
  
 **Дополнительные области фреймов**  
 |Параметр|Описание|  
|--------------|--------------|  
|**Закрепляемая область обозревателя**|Создание закрепляемой области, аналогичной области **Обозреватель решений** Visual Studio, слева от фрейма главного окна.|  
|**Закрепляемый фрейм вывода**|Создание закрепляемой области, аналогичной области **Вывод** Visual Studio, ниже фрейма главного окна.|  
|**Закрепляемая область свойств**|Создание закрепляемой области, аналогичной области **Свойства** Visual Studio, справа от фрейма главного окна.|  
|**Область навигации**|Создание закрепляемой области, аналогичной панели переходов Outlook, слева от фрейма главного окна.|  
|**Заголовок окна**|Создание заголовка окна, аналогичного Office, выше фрейма главного окна.|  
  
 **Число файлов в текущем списке файлов**  
 Указывает число файлов, которые будут включены в список последних использовавшихся файлов.  Значение по умолчанию — 4.
