# <a name="application-type-mfc-application-wizard"></a>Тип приложения, мастер приложений MFC
Используйте эту страницу [мастер приложений MFC](../../mfc/reference/mfc-application-wizard.md) для проектирования и добавления базовых функций нового приложения MFC.  
  
 **Тип приложения**  
 Указывает тип поддержки документов, который вы хотите создать в приложении. Тип выбранного приложения определяет параметры пользовательского интерфейса, доступных для вашего приложения. В разделе [возможности пользовательского интерфейса, мастер приложений MFC](../../mfc/reference/user-interface-features-mfc-application-wizard.md) подробнее.  
  
 Дополнительные сведения о типах документов см.  
  
-   [SDI и MDI](../../mfc/sdi-and-mdi.md)  
  
-   [Окна фрейма](../../mfc/frame-windows.md)  
  
-   [Классы окна фрейма](../../mfc/frame-window-classes.md)  
  
-   [Документы, представления и платформа](../../mfc/documents-views-and-the-framework.md)  
  
-   [Диалоговые окна](../../mfc/dialog-boxes.md)  
  
|Параметр|Описание|  
|------------|-----------------|  
|**Один документ**|Создание архитектуры однооконный интерфейс (SDI) для приложения с классом просмотра, основанным на [CView-класс](../../mfc/reference/cview-class.md). Базовый класс для просмотра можно изменить [создаваемые классы, мастер приложений MFC](../../mfc/reference/generated-classes-mfc-application-wizard.md) мастера. Для создания приложения на основе форм, например, использовать [CFormView Class](../../mfc/reference/cformview-class.md) для класса представления.<br /><br /> В этом типе приложения окно фрейма документа может содержать только один документ.|  
|**Несколько документов**|Создание архитектуры интерфейса (MDI) документа для приложения с классом просмотра, основанным на `CView`. Базовый класс для просмотра можно изменить **классы, создаваемые** мастера. Для создания приложения на основе форм, например, использовать `CFormView` для класса представления.<br /><br /> В этом типе приложения окно фрейма документа может содержать несколько дочерних окон.|  
|**Документы с вкладками**|Каждый документ размещается на отдельной вкладке.|  
|**На базе диалогового окна**|Создание архитектуры диалогового окна для приложения с классом диалогового окна, основанным на `CDialog`. (Для создания диалогового окна HTML, выберите поле **использовать диалоговое окно HTML**.)|  
|**Использовать диалоговое окно HTML**|Диалоговое окно поле только для приложений. Производный класс диалогового окна из [класс CDHtmlDialog](../../mfc/reference/cdhtmldialog-class.md) вместо [класс CDialog](../../mfc/reference/cdialog-class.md). Если этот флажок установлен, `CDHtmlDialog` перечисленные в **базового класса** поле [создаваемые классы, мастер приложений MFC](../../mfc/reference/generated-classes-mfc-application-wizard.md) мастера.<br /><br /> A `CDHtmlDialog`-диалоговое окно производном отображаются диалоговые окна HTML, осуществляется обмен данными с HTML, элементы управления и обрабатываются события HTML.|  
|**Несколько документов верхнего уровня**|Создание архитектуры верхнего уровня для приложения с классом просмотра, основанным на `CView`.<br /><br /> В этот тип приложения, когда пользователь щелкает **New** (или **новый кадр**) на **файл** меню, приложение создает окно, которое неявным образом является дочерним рабочего стола. Фрейм нового документа появляется в панели задач и не ограничена клиентской области окна приложения.|  
  
 **Поддержка архитектуры "документ представление"**  
 Указывает, следует ли включить архитектура документов и представлений в приложение с помощью [CDocument-класс](../../mfc/reference/cdocument-class.md) и [CView-класс](../../mfc/reference/cview-class.md) (по умолчанию). Снимите этот флажок, при переносе приложения не MFC или необходимо уменьшить размер скомпилированных исполняемых файлов. По умолчанию для приложения без архитектуры документа/представления является производным от [класс CWinApp](../../mfc/reference/cwinapp-class.md), и не включает поддержку MFC для открытия документа из файла на диске.  
  
 **Язык ресурсов**  
 Задает язык ресурсов. Список языков, доступных в системе, установки Visual Studio. Если вы хотите выбрать язык, отличный от языка системы, соответствующую папку с шаблонами для этого языка должна быть установлена. Дополнительные сведения об установке языковых ресурсов, отличается от значения по умолчанию, доступных в **язык ресурсов** список см. в разделе [поддержка мастера для других языков](../../ide/wizard-support-for-other-languages.md).  
  
 Язык, который выбран отражается в **локализованные строки** параметр [строки шаблонов документов, мастер приложений MFC](../../mfc/reference/document-template-strings-mfc-application-wizard.md) мастера.  
  
 **Использовать библиотеки Юникод**  
 Указывает, следует ли использовать Юникод или Юникод версии библиотеки MFC.  
  
 **Стиль проекта**  
 Указывает, имеет ли приложение Стандартная MFC, проводник, Visual Studio или архитектуры Office и отображения. Дополнительные сведения см. в разделе [Создание приложения MFC в стиле проводника файл](../../mfc/reference/creating-a-file-explorer-style-mfc-application.md).  
  
|Параметр|Описание|  
|------------|-----------------|  
|**Стандартный стиль MFC**|Предоставляет стандартную архитектуру приложения MFC.|  
|**Проводник файлов**|Реализует файлов приложений в стиле обозревателя с помощью окна-разделителя, где левая панель – [класс CTreeView](../../mfc/reference/ctreeview-class.md) и правая панель – [CListView-класс](../../mfc/reference/clistview-class.md).|  
|**Visual Studio**|Реализует приложение Visual Studio, таким как, четырех закрепляемых областей (**представление файлов**, **представление классов**, **свойства**, и **вывода**), являющиеся производными от [класса CDockablePane](../../mfc/reference/cdockablepane-class.md) и фрейма главного окна, который является производным от [CMDIFrameWndEx Class](../../mfc/reference/cmdiframewndex-class.md) (по умолчанию).|  
|**Office**|Реализует Office, таким как приложения, содержащего ленты, который является производным от [класса CMFCRibbonBar](../../mfc/reference/cmfcribbonbar-class.md), панель Outlook, который является производным от [класса CMFCOutlookBar](../../mfc/reference/cmfcoutlookbar-class.md), строки заголовка, который является производным от [класса CMFCCaptionBar](../../mfc/reference/cmfccaptionbar-class.md)и главного фрейма, который является производным от [CMDIFrameWndEx Class](../../mfc/reference/cmdiframewndex-class.md).|  
  
 **Визуальный стиль и цвета**  
 Определение визуального стиля приложения. Доступны следующие параметры.  
  
-   **По умолчанию Windows машинный код**  
  
-   **Office 2003**  
  
-   **Visual Studio 2005**  
  
-   **Office 2007 (синяя тема)**  
  
-   **Office 2007 (черный тема)**  
  
-   **Office 2007 (серебристая тема)**  
  
-   **Office 2007 (голубой цвет темы)**  
  
 **Разрешить смену визуального стиля**  
 Указывает ли пользователь может изменить визуальный стиль приложения во время выполнения, обычно путем выбора соответствующего стиля в меню или на ленте.  
  
 **Использование MFC**  
 Указывает, как связь с библиотекой MFC. По умолчанию библиотека MFC привязана как общую библиотеку DLL.  
  
|Параметр|Описание|  
|------------|-----------------|  
|**Использовать MFC в общей библиотеке DLL**|Ссылки для приложения в качестве общей библиотеки DLL библиотеки MFC. Приложение вызывает библиотеку MFC во время выполнения. Этот параметр уменьшает требования к объему дискового пространства и ОЗУ приложения, которые состоят из нескольких исполняемых файлов, использующих библиотеку MFC. Приложения Win32 и MFC могут вызывать функции библиотеки DLL (по умолчанию)|  
|**Использовать MFC в статической библиотеке**|Привязка приложения к статической библиотеке MFC во время построения.|
