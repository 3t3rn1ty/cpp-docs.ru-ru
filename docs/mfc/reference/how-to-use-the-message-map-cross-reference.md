# Практическое руководство. Использование перекрестной ссылки схемы сообщений
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

В меткаых \<memberFxn\>записях, разработайте свой функцией\-членом производного класса [CWnd](../Topic/CWnd%20Class.md).  Присвойте функции любое имя необходимо.  Другие функции, такие как `OnActivate`, функции\-члены класса `CWnd`.  Если вызывается, передаваемого сообщения в функции Windows `DefWindowProc`.  В отростчатым сообщения уведомлений Windows, переопределите соответствующие функции `CWnd` в производном классе.  Функция должна вызывать переопределенная функции в базовом классе, чтобы позволить базовому классу и Windows ответа на сообщение.  
  
 Во всех случаях поместить прототип функции в `CWnd`\- заголовок производного класса и код запись сопоставления сообщений, как показано в примере.  
  
 Следующие термины используются:  
  
|Термин|Определение|  
|------------|-----------------|  
|id|Любое определенное пользователем идентификатор пункта меню сообщения \(**WM\_COMMAND** \) или идентификатор элемента управления \(сообщения уведомления дочернего окна\).|  
|«сообщение» и «wNotifyCode»|Идентификатор сообщения Windows, как определено в WINDOWS.H.|  
|nMessageVariable|Имя переменной, содержащей возвращаемое значение в функции Windows **RegisterWindowMessage**.|
