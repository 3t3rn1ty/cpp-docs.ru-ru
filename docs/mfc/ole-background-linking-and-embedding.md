# <a name="ole-background-linking-and-embedding"></a>Поддержка OLE. Связь и внедрение
С помощью команды Вставить в приложение контейнера можно создать интегрированный компонент или встроенного элемента. Исходные данные для внедренного элемента хранится как часть документа OLE, в которой он содержится. Таким образом файл документа для документа текстовый процессор может содержать текст и также может содержать растровые изображения, диаграммы, формулы или любого другого типа данных.  
  
 OLE предоставляет еще один способ включить данные из другого приложения: Создание связанного компонента или связанный элемент или ссылку. Шаги по созданию связанного элемента, аналогичны для создания встроенного элемента, за исключением того, можно связать команды вместо команды вставки. В отличие от внедренного компонента связанный компонент сохраняет путь к исходным данным, который часто в отдельном файле.  
  
 Например если вы работаете в текстовом документе процессора и создать связанный элемент, чтобы некоторые ячейки листа, данных для связанного элемента хранится в исходном документе электронной таблицы. Текстовый документ содержит только данные, указав, где элемент можно найти, то есть, он содержит ссылку на исходный документ электронной таблицы. При двойном щелчке ячейки, запускается приложение электронной таблицы и исходный документ электронная таблица загружается из которой он хранился.  
  
 Каждый элемент OLE внедренные или связанные, имеет ли тип, связанный с ним зависимости от приложения, в которой он был создан. Например элемент кисти Microsoft является один тип элемента и элемента Microsoft Excel — еще один тип. Однако некоторые приложения, можно создать более чем один тип элемента. Например Microsoft Excel можно создать элементы листа, элементов диаграммы и macrosheet элементов. Каждый из этих элементов можно однозначно идентифицировать системой с помощью идентификатора класса или **CLSID**.
