# <a name="command-routing-illustration"></a>Демонстрация маршрутизации команд
Чтобы продемонстрировать, рассмотрим сообщение команды из очистить все пункту меню в меню "Правка" MDI-приложения. Предположим, что функция обработчика для данной команды происходит с быть функцией-членом класса документа приложения. Вот, как команда достигнет его обработчика, после выбора элемента меню.  
  
1.  Главное окно получает сообщение, команда сначала.  
  
2.  Главное окно рамки MDI позволяет дочернее окно текущего активного интерфейса MDI самостоятельно обработать команду.  
  
3.  Стандартная Маршрутизация дочернего окна фрейма MDI позволяет представление его в командной строке перед проверкой собственную схему сообщений.  
  
4.  Представление сначала проверяет собственную схему сообщений и поиск нет обработчика, затем направляет команду его связанный документ.  
  
5.  Документ проверяет схему сообщений и находит обработчик. Вызывается эта функция-член документа и маршрутизации останавливается.  
  
 Если документ не содержит обработчик, он будет перенаправить команду рядом с его шаблоном документа. Затем команда вернет представление, а затем фрейм окна. Наконец окно фрейма возвращают схему сообщений. Если этого не удалось выполнить проверку также, команда будет перенаправлен обратно в главное окно рамки MDI, а затем объект приложения — конечного назначения необработанных команд.
