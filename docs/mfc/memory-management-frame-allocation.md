# <a name="memory-management-frame-allocation"></a>Управление памятью: выделение кадров
Выделение во фрейме планка «кадра стека», заданный при каждом вызове функции. Кадр стека — это область памяти, временно содержит аргументы для функции, а также все переменные, определенные локальным для данной функции. Переменные часто называются «automatic» переменные, поскольку компилятор автоматически выделяет место для их.  
  
 Существует две ключевые характеристики выделения кадра. Во-первых при определении локальной переменной, достаточно места выделяется в кадре стека для хранения всего переменной, даже если это большой массив или структуры данных.  Для переменных локальной функции этого перехода области происходит, когда функция завершает работу, но области переменной фрейма может быть меньше, чем функция, если используются вложенные скобки. Очень важно автоматического удаления переменных фрейма. В случае простые типы-примитивы (такие как `int` или **байтов**), массивов и структур данных, автоматическое удаление просто освобождает память, используемую в переменной. Так как переменная вышло за пределы области, он не доступен несмотря на это. В случае объектов C++ процесс автоматического удаления является более сложным.  
  
 Если объект определен как переменной фрейма, его конструктор вызывается автоматически в той точке, где встречается определение. Когда объект выходит за пределы области, его деструктор вызывается автоматически до освобождения памяти для объекта. Это автоматическое создание и уничтожение могут быть очень удобны, но следует иметь в виду автоматического вызовов, особенно деструктор.  
  
 Ключевым преимуществом выделение объектов в кадре — что они будут автоматически удалены. При распределении объектов в кадре, не нужно волноваться по поводу утраченный набор объектов, причиной утечки памяти. (Подробнее о утечки памяти, см. в статье [обнаружение утечек памяти в MFC](http://msdn.microsoft.com/en-us/29ee8909-96e9-4246-9332-d3a8aa8d4658).) Выделение кадров недостаток заключается в, переменные не может использоваться за пределами их области. Другим фактором при выборе выделение кадров и выделение кучи является, что для больших структур и объектов, зачастую лучше использовать кучи вместо стека хранилища, так как пространство стека ограничено часто.
