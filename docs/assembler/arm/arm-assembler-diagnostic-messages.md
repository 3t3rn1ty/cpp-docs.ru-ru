# ARM Assembler Diagnostic Messages
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

Ассемблер ARM корпорации Майкрософт \(*armasm*\) выдает диагностических предупреждений и ошибок при их обнаружении.  Статья описывает наиболее часто обнаружил сообщения.  
  
## Синтаксис  
  
```  
  
filename(lineno) : [error|warning] Anum: message  
```  
  
## Диагностические сообщения  
  
### Ошибки  
 A2193: Эта инструкция приводит к непредсказуемому поведению  
 Архитектура ARM не гарантирует, что произойдет при выполнении данной инструкции.  Дополнительные сведения о строго определенные формы этой инструкции обратитесь к [Справочном руководстве для архитектуры ARM](http://go.microsoft.com/fwlink/?LinkId=246464).  
  
```  
  
ADD r0, r8, pc         ; A2193: this instruction generates unpredictable behavior  
  
```  
  
 A2196: инструкция не может быть закодирован в 16 бит  
 Указанной инструкции не может быть закодирован как 16\-разрядных инструкций Thumb.  Укажите 32\-разрядных инструкций или изменить код для приведения целевой метки в диапазоне 16\-разрядных инструкций.  
  
 Ассемблер могут пытаться кодировать ветви в 16 бит и этой ошибкой, несмотря на то, что 32\-разрядные ветви безопасно\-кодируемыми.  Эту проблему можно решить с помощью `.W` спецификатор явно пометить ветвь как 32\-разрядные.  
  
```  
  
  ADD.N r0, r1, r2      ; A2196: instruction cannot be encoded in 16 bits  
  
  B.W label             ; OK  
  B.N label             ; A2196: instruction cannot be encoded in 16 bits  
  SPACE 10000  
label  
  
```  
  
 A2202: Pre\-Ы инструкции синтаксиса не допускается в области ползунка полосы ПРОКРУТКИ  
 Thumb кода необходимо использовать синтаксис единого языка ассемблера \(Ы\).  Старый синтаксис больше не принимаются  
  
```  
  
ADDEQS r0, r1         ; A2202: Pre-UAL instruction syntax not allowed in THUMB region  
ADDSEQ r0, r1         ; OK  
  
```  
  
 A2513: Поворот должен быть даже  
 В режиме ARM имеется альтернативный синтаксис для определения констант.  Вместо `#<const>`, можно написать `#<byte>,#<rot>`, который представляет константу, которая получается, повернув значение `<byte>` по `<rot>`.  При использовании этого синтаксиса необходимо сделать значение `<rot>` даже.  
  
```  
  
MOV r0, #4, #2       ; OK  
MOV r0, #4, #1       ; A2513: Rotation must be even  
  
```  
  
 A2557: Неверное число байтов для записи назад  
 Неоновый СВЕТ структуры загружать и сохранять инструкции \(`VLDn`, `VSTn`\), не существует альтернативный синтаксис для указания обратной записи для базового регистра.  Вместо размещения восклицательный знак \(\!\) после адреса, можно указать непосредственное значение, указывающее смещение для добавления базового регистра.  Если использовать этот синтаксис, необходимо указать точное число байтов, которые были загружены или хранимой инструкцией.  
  
```  
  
VLD1.8 {d0-d3}, [r0]!         ; OK  
VLD1.8 {d0-d3}, [r0], #32     ; OK  
VLD1.8 {d0-d3}, [r0], #100    ; A2557: Incorrect number of bytes to write back  
  
```  
  
### Предупреждения  
 A4228: Значение выравнивания превышает выравнивание области; не гарантируется выравнивание  
 Выравнивание, которое указано в `ALIGN` директива больше, чем выравнивание, включающего `AREA`.  Как следствие, ассемблер не может гарантировать, что `ALIGN` директивы будут удовлетворяться.  
  
 Чтобы устранить эту проблему, можно указать на `AREA` директивы `ALIGN` атрибут, который равен или больше, чем нужное выравнивание.  
  
```  
  
AREA |.myarea1|  
ALIGN 8           ; A4228: Alignment value exceeds AREA alignment; alignment not guaranteed  
  
AREA |.myarea2|,ALIGN=3  
ALIGN 8           ; OK  
  
```  
  
 A4508: Рекомендуется использование этой константы \(повернуто\)  
 В режиме ARM имеется альтернативный синтаксис для определения констант.  Вместо `#<const>`, можно написать `#<byte>,#<rot>`, который представляет константу, которая получается, повернув значение `<byte>` по `<rot>`.  В некоторых контекстах ARM рекомендуется использовать эти константы \(повернуто\).  В этих случаях использовать базовый `#<const>` синтаксис вместо.  
  
```  
  
ANDS r0, r0, #1                ; OK  
ANDS r0, r0, #4, #2            ; A4508: Use of this rotated constant is deprecated  
  
```  
  
 A4509: В этой форме условной инструкции является устаревшим  
 Эта форма условной инструкции были признаны ARM ARMv8 архитектуры.  Рекомендуется изменить код, чтобы использовать условные ветви.  Чтобы увидеть, какие условные инструкции по\-прежнему поддерживаются, обратитесь к [Справочном руководстве для архитектуры ARM](http://go.microsoft.com/fwlink/?LinkId=246464).  
  
 Это предупреждение не выдаются при  `- oldit` использовать параметр командной строки.  
  
```  
  
ADDEQ r0, r1, r8              ; A4509: This form of conditional instruction is deprecated  
  
```
