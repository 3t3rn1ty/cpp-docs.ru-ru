# Общие сведения о трансляции файлов
[!INCLUDE[vs2017banner](../assembler/inline/includes/vs2017banner.md)]

Программы на языке C\+\+ подобно программам на языке C состоят из одного или нескольких файлов.  Каждый из этих файлов преобразуется в указанном ниже концептуальном порядке \(реальный порядок соответствует правилу "как если бы": преобразование должно происходить так, как если бы соблюдался следующий порядок действий\).  
  
1.  Лексический анализ.  На этом этапе преобразования сопоставляются символы, обрабатываются триграфы, соединяются строки и выполняется разбивка на лексемы.  
  
2.  Предварительная обработка.  На этом этапе преобразования вводятся вспомогательные исходные файлы, на которые ссылается директива `#include`, обрабатываются директивы создания строки и создания символа и выполняются вставка токена и расширение макроса \(дополнительные сведения см. в разделе [Директивы препроцессора](../preprocessor/preprocessor-directives.md) *Справочника по препроцессору*\).  Результат этапа предварительной обработки — последовательность токенов, которые в совокупности определяют "запись преобразования".  
  
     Директивы препроцессора всегда начинаются со знака решетки \(**\#**\) \(т. е. первым отличным от проблема символом в строке должен быть знак решетки\).  В строке может присутствовать только одна директива препроцессора.  Например:  
  
    ```  
    #include <iostream>  // Include text of iostream in   
                         //  translation unit.  
    #define NDEBUG       // Define NDEBUG (NDEBUG contains empty   
                         //  text string).  
    ```  
  
3.  Создание кода.  На этом этапе преобразования токены, созданные на этапе предварительной обработки, используются для создания объектного кода.  
  
     Выполняется синтаксическая и семантическая проверка исходного кода.  
  
 Дополнительные сведения см. в разделе [Этапы преобразования](../preprocessor/phases-of-translation.md) *Справочника по препроцессору*.  
  
 Препроцессор C\+\+ — это строгое надмножество препроцессора ANSI C, но имеет ряд отличий.  Ниже указано несколько различий между препроцессорами ANSI C и C\+\+.  
  
-   Поддерживаются однострочные комментарии.  Дополнительные сведения см. в разделе [Комментарии](../cpp/comments-cpp.md).  
  
-   Один предустановленный макрос \(**\_\_cplusplus**\) определен только для C\+\+.  Дополнительные сведения см. в разделе [Предустановленные макросы](../preprocessor/predefined-macros.md) *Справочника по препроцессору*.  
  
-   Препроцессор C не распознает следующие операторы C\+\+: **.\***, **–\>\*** и `::`.  Дополнительные сведения об операторах см. в разделах [Операторы](../cpp/cpp-built-in-operators-precedence-and-associativity.md) и [Выражения](../cpp/expressions-cpp.md).
