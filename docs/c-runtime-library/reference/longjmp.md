# <a name="longjmp"></a>longjmp

Восстанавливает среду стека и языковой стандарт выполнения.

## <a name="syntax"></a>Синтаксис

```C
void longjmp(
   jmp_buf env,
   int value
);
```

### <a name="parameters"></a>Параметры

*env* переменной, в которой среда хранится.

*значение* значение, возвращаемое для **setjmp** вызова.

## <a name="remarks"></a>Примечания

**Longjmp** функция восстанавливает языкового стека среде и выполнения ранее сохраненных в *env* по **setjmp**. **setjmp** и **longjmp** предоставляют способ выполнения нелокальные **goto**; они обычно используются для передачи управления выполнением в код обработки ошибок или восстановления в вызванной ранее подпрограмме без с помощью обычных вызовов и возврата соглашения.

Вызов **setjmp** вызывает текущую среду стека должен быть сохранен в *env*. В последующем вызове **longjmp** восстанавливает сохраненную среду и возвращает управление в точке сразу после соответствующего **setjmp** вызова. Возобновляет выполнение как если бы *значение* просто вернула по **setjmp** вызова. Значения переменных (за исключением переменные регистра), доступные для получившей управление подпрограммы содержат значения, они имели при **longjmp** был вызван. Значения регистровых переменных непредсказуемы. Значение, возвращаемое **setjmp** должны быть ненулевыми. Если переданный параметр *value* равен 0, фактическое возвращаемое значение будет равно 1.

Вызовите **longjmp** перед функции, вызвавшей **setjmp** возвращает; в противном случае результаты будут непредсказуемыми.

Обратите внимание на следующие ограничения при использовании **longjmp**:

- Не рассчитывайте на то, что значения регистровых переменных останутся теми же. Значения переменных регистра в сопоставление, вызывающем **setjmp** не могут быть восстановлены после нужные значения **longjmp** выполняется.

- Не используйте **longjmp** для передачи управления из подпрограммы обработки прерываний, пока прерывание вызвано путем исключения с плавающей запятой. В этом случае программа может возвратить из обработчика прерываний через **longjmp** если он сначала повторно инициализирует пакета вычисления с плавающей запятой, вызвав **_fpreset**.

     **Примечание** соблюдать осторожность при использовании **setjmp** и **longjmp** в программах на C++. Так как эти функции не поддерживают семантику объектов C++, лучше использовать механизм обработки исключений C++.

Дополнительные сведения см. в разделе [Использование setjmp и longjmp](../../cpp/using-setjmp-longjmp.md).
