# mbrtoc16, mbrtoc32
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

Преобразует первый многобайтовый символ в строке обычных символов в эквивалентный символ в кодировке UTF\-16 или UTF\-32.  
  
## Синтаксис  
  
```  
size_t mbrtoc16(   
   char16_t* destination,   
   const char* source,   
   size_t max_bytes,   
   mbstate_t* state   
);  
  
size_t mbrtoc32(  
   char32_t* destination,   
   const char* source,   
   size_t max_bytes,   
   mbstate_t* state   
);  
  
```  
  
#### Параметры  
 `destination`  
 Указатель на эквивалент `char16_t` или `char32_t` многобайтового символа, который нужно преобразовать. Если значение равно null, функция не сохраняет значение.  
  
 `source`  
 Указатель на строку многобайтовых символов, которую нужно преобразовать.  
  
 `max_bytes`  
 Максимальное число байтов в `source`, которое нужно проанализировать на наличие символа для преобразования. Это значение должно быть в диапазоне от одного до количества байтов \(включая символ завершения null\), оставшегося в `source`.  
  
 `state`  
 Указатель на объект состояния преобразования `mbstate_t`, используемый для преобразования многобайтовой строки в один или несколько выходных символов.  
  
## Возвращаемое значение  
 В случае успешного выполнения возвращает значение, соответствующее первому из приведенных ниже условий, с учетом текущего значения `state`.  
  
|Значение|Условие|  
|--------------|-------------|  
|0|Следующие `max_bytes` или меньше символов, преобразованных из `source`, соответствуют расширенному символу null, то есть значению, сохраняемому в случае, если `destination` не равно null.<br /><br /> `state` содержит начальное состояние сдвига.|  
|От 1 до `max_bytes` включительно|Возвращаемое значение равно количеству байтов в `source`, составляющих допустимый многобайтовый символ. Если `destination` не равно null, сохраняется преобразованный расширенный символ.|  
|\-3|Следующий расширенный символ, полученный в результате предыдущего вызова функции, сохраняется в `destination`, если `destination` не равно null. Байты из `source` не используются при этом вызове функции.<br /><br /> Если `source` указывает на многобайтовый символ, для представления которого требуется несколько расширенных символов, например на суррогатную пару, значение `state` обновляется так, что при следующем вызове функции записывается дополнительный символ.|  
|\-2|Следующие `max_bytes` байтов представляют неполный, но потенциально допустимый многобайтовый символ. Значение в `destination` не сохраняется. Этот результат может быть получен, если `max_bytes` равно нулю.|  
|\-1|Произошла ошибка кодирования. Следующие `max_bytes` или менее байт не составляют полный и допустимый многобайтовый символ. Значение в `destination` не сохраняется.<br /><br /> `EILSEQ` сохраняется в `errno`, а состояние преобразования `state` не указано.|  
  
## Заметки  
 Функция `mbrtoc16` считывает до `max_bytes` байтов из `source`, чтобы найти первый полный допустимый многобайтовый символ, а затем сохраняет эквивалентный символ в кодировке UTF\-16 в `destination`. Исходные байты интерпретируются в соответствии с многобайтовым языковым стандартом текущего потока. Если для представления многобайтового символа требуется несколько выходных символов в кодировке UTF\-16, например в случае с суррогатной парой, то значение `state` задается так, чтобы следующий символ UTF\-16 сохранялся в `destination` при следующем вызове `mbrtoc16`. Функция `mbrtoc32` идентична, но выходные данные сохраняются в виде символа в формате UTF\-32.  
  
 Если `source` имеет значение null, эти функции возвращают результат, эквивалентный вызову с использованием аргументов `NULL` для `destination`, `""` для `source` и `1` для `max_bytes`. Значения, передаваемые в параметрах `destination` и `max_bytes`, игнорируются.  
  
 Если `source` не равно null, функция начинает обработку с начала строки и проверяет не более `max_bytes` байт для определения числа байтов, необходимых для составления следующего многобайтового символа, включая любые последовательности сдвигов. Если проверяемые байты содержат допустимый и полный многобайтовый символ, функция преобразует его в эквивалентный 16\- или 32\-битовый расширенный символ \(либо несколько символов\). Если `destination` не равно null, функция сохраняет первый \(и, возможно, единственный\) итоговый символ в конечном объекте. Если требуются дополнительные выходные символы, значение в `state` задается так, чтобы при последующих вызовах функции выводились дополнительные символы и возвращалось значение –3. Если дополнительных символов больше не требуется, `state` устанавливается в начальное состояние сдвига.
